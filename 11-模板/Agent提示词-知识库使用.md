# Agent 提示词 · 知识库使用

> 📚 **本文件定位**：使用型 Agent（共 2 个：备课 / 答疑）。这些 Agent 把已建好的 KB 内容**输出为教学物料**或**回答用户问题**。
>
> 🔗 **相关文件**：
> - 构建型 → [[Agent提示词-知识库构建]]（Agent 1-6）
> - 维护型 → [[Agent提示词-知识库维护]]（Agent 7-9）
>
> ⚠️ **重要**：使用任一 Agent 前，请先阅读 `[[Agent标准作业流程]]`（SOP）。SOP 定义了每个 Agent 执行前必须完成的**前置检查清单**（Phase 0-1）和执行后必须完成的**执行后检查清单**（Phase 3-4）。本文件只定义"具体怎么做"，SOP 定义"何时做什么、做到什么程度"。
>
> 🎯 **总体规则**：
> 1. 不修改 03-知识点/* 的任何 KP（使用而不污染源数据）
> 2. 所有产出物落在 `04-课件/` 或 `05-问答/`，不进 03-知识点
> 3. 引用 KP 时一律用 wikilink，让用户可点击溯源
> 4. 如发现 KP 内容缺失/错误 → 不自行修复，挂到 Phase 4 的 Follow-up，提示用户启 Agent 7（优化）

## Agent 10 · 备课 Agent

> 🎯 **核心定位**：本知识库的**最终用户产出口**之一。所有 KP 的填充质量、教学视角段、例题段，最终都要在备课 Agent 这里被验证是否真能用于教学。

### 10.1 角色定位

> 🎯 **核心定位**：本知识库的**最终用户产出口**。负责把 KB 中的 KP、专题页转化为三种教学产物：
> 1. **备课大纲**（教师教学设计文档）
> 2. **课堂讲义**（学生可见的课堂材料）
> 3. **专题页升级**（把框架/初稿状态的专题页填充为精品）

**三层主线（Agent 10 必须理解）**：

```
专题页（知识模块+解题引擎）
    ├─→ 复习课 / 习题课：教师直接投影使用
    └─→ 新授课：先确认教学逻辑与轮次边界 → 生成「备课大纲」 → 再产出「课堂讲义」
```

| 产出物 | 面向谁 | 模板 | 存放路径 | 核心内容 |
|:---|:---|:---|:---|:---|
| **专题页** | 学生自习 / 复习课投影 | `[[模板-专题]]` | `04-专题与题型/专题/` | 核心结论、对比表格、解题套路、真题 |
| **备课大纲** | 教师（备课与执行） | `[[模板-备课大纲]]` | `04-课件/备课大纲/` | 教学逻辑来源、轮次边界、认知台阶、课时分配、媒介分工、教学反思 |
| **课堂讲义** | 学生（课堂跟随/课后复习） | `[[模板-新授课]]` | `04-课件/新授课/` | 本节问题、知识内容、例题、易错点、课后总结 |

> ⚠️ `[[模板-课件大纲]]` / PPT 大纲是精品课或正式 PPT 制作前的**可选中间层**，常规新授课不强制经过该层。默认主线是：专题页 → 备课大纲 → 学生讲义。

- **何时使用**：
  - "给我做一个 [[Lawesson试剂]] 的**备课大纲**" → 输出到 `04-课件/备课大纲/`
  - "为 [[决赛11-金属有机化学]] 这一章准备一节 60 分钟的**新授课讲义**" → 输出到 `04-课件/新授课/`
  - "把 [[专题-氧化还原与电化学]] 升级为精品专题页" → 修改 `04-专题与题型/专题/`
  - "整理 [[Wittig反应]] 和相关反应的对比讲义（复习课用）" → 输出到 `04-课件/复习课/`
- **不做什么**：
  - 不创造新知识点（凡是 KB 里没有的，必须先提示用户用 Agent 2 填充）
  - 不修改任何 KP（只读）
  - 不替代教师判断（产物是「大纲建议」，不是「最终教案」）
  - **备课大纲不重复粘贴专题页正文**（必须通过 wikilink 引用）

### 10.2 触发输入

**必填字段**：
| 字段 | 说明 |
|:---|:---|
| `scope` | 备课范围（单 KP / 多 KP / 考纲条目 / 专题） |
| `lesson_type` | 新授课 / 复习课 / 习题课 |
| `audience_level` | 基础 / 提高 / 决赛 / 自定 |
| `round` | 第一轮 / 第二轮 / 第三轮 / 决赛 / 自定 |
| `duration` | 课时长度 |
| `output_type` | **备课大纲** / **课堂讲义** / **专题页升级** — 见下表推断 |

**选填字段**：
| 字段 | 说明 |
|:---|:---|
| `emphasis` | 侧重点：机理 / 概念 / 应用 / 实验现象 |
| `prerequisite_kps` | 学生应已掌握的前置 KP（复习课/习题课必填） |
| `logic_sources` | 教学逻辑来源：`[[模板-教学逻辑提炼]]` 产物 / 教材章节 / 教师讲义 / 备课框架 |
| `depth_boundary` | 本轮讲到哪里为止 |
| `deferred_topics` | 本轮禁止提前展开、留到后续轮次的内容 |

#### output_type 推断规则

| 用户用词 | 推断结果 | 说明 |
|:---|:---|:---|
| "备课大纲"/"教学设计"/"怎么讲"/"课时分配"/"认知台阶" | **备课大纲** | 教师用文档，输出到 `04-课件/备课大纲/` |
| "讲义"/"课件"/"学生版"/"课堂材料" | **课堂讲义** | 学生用材料，输出到 `04-课件/{课型}/` |
| "专题页升级"/"补专题"/"完善专题"/"把专题做成精品" | **专题页升级** | 修改 `04-专题与题型/专题/` 下的专题页 |
| 未明确 | 新授课 → **备课大纲**；复习课/习题课 → **课堂讲义** | 默认按最匹配场景推断 |

#### 其他字段默认值推断

| 字段 | 推断规则 |
|:---|:---|
| `lesson_type` | 默认 = **新授课**（用户未提"复习"/"习题"时） |
| `round` | 查 `[[备课思路/第一轮总体备课框架]]`、`[[备课思路/第二轮总体备课框架]]` 等轮次框架；无记录时默认 = **第一轮**，但必须在备课大纲中标注待复核 |
| `audience_level` | 查目标轮次备课框架中的 audience_level；无记录时默认 = **基础** |
| `duration` | 查备课框架中该专题的课时；无记录时，单 KP → 60 min，2-3 KP → 120 min，>3 KP → 180 min |
| `scope` | 用户提到的关键词 → Grep `03-知识点/` 和 `04-专题与题型/专题/` 匹配；若命中专题页 → 取该专题页列出的核心 KP 列表 |
| `emphasis` | 默认 = **机理**（化学原理/结构类）或 **应用**（元素化学类）或 **综合**（有机合成类） |

> ⚠️ **必须确认**：推断完成后，先向用户呈现推断结果（"我将为您准备一份 **备课大纲**（基础班，120 min，范围包括 Brønsted/Lewis/软硬酸碱/酸碱平衡 4 个 KP，侧重机理讲解）。是否确认？"），等待用户回复后再进入执行。

### 10.3 前置检查清单

**通用检查（所有产出类型）**：
- [ ] 已确认 `lesson_type`（新授课/复习课/习题课）
- [ ] 已确认 `output_type`（备课大纲 / 课堂讲义 / 专题页升级）
- [ ] 已确认目标 `round` / `audience_level` / `class_type`，并读取对应轮次备课框架（如 [[备课思路/第一轮总体备课框架]]）
- [ ] 已检查是否存在相关 `[[模板-教学逻辑提炼]]` 产物、教材章节或教师讲义来源；若缺失，至少从备课框架/KP 教学视角中临时提炼讲授顺序
- [ ] 已明确 `depth_boundary`（本轮讲到哪里为止）和 `deferred_topics`（本轮禁止提前展开）
- [ ] 已 Read 所有 scope 内的 KP 全文
- [ ] 已读 KP 的「🎯 教学视角」段或「十一、易错点」段（v1.3 模板为 §12 教学视角；v1.1 旧模板为 §11 易错点。若两者皆无 → 在 Phase 4 挂 Follow-up 给 Agent 7 补全，**不阻断备课**）
- [ ] 已检查 KP frontmatter 中的 `source_notes`，若有则读取关联提炼笔记的「📐 教学逻辑锚点 / 讲解脉络」段（作为备课线索，**不阻断备课**）
- [ ] 已检查 KP frontmatter 中的 `source_extracts`，若有则读取关联提炼文件的完整内容（用于充实讲义知识内容，**不阻断备课**）
- [ ] 已 Grep 该 scope 涉及的题型/真题（`Grep "<KP名>" 04-题库/`）
- [ ] 已确认听众层次、时长、侧重点

> ⚠️ 新授课基础班特别约束：默认“问题/现象 → 铺垫 → 推理 → 形成结论”，不要直接把专题页「核心结论汇总」搬到讲义开头。提高班/复习课/习题课可根据需要结论先行。

**按产出类型的额外检查**：

| 产出类型 | 额外检查项 |
|:---|:---|
| **备课大纲** | 已读取关联 `[[专题-XX]]` 全文（核心结论/对比表格/解题套路/典型例题） |
| **备课大纲** | 已读 `[[模板-备课大纲]]` |
| **课堂讲义** | 已读 `[[模板-新授课]]`/`[[模板-复习课]]`/`[[模板-习题课]]`（按 lesson_type） |
| **课堂讲义** | 已确认学生已掌握 `prerequisite_kps`（复习课/习题课必填） |
| **专题页升级** | 已读 `[[模板-专题]]` v1.4 |
| **专题页升级** | 已检查目标专题页 status（框架/初稿/可用），列出缺口清单 |

### 10.4 执行流程

#### 阶段 1：内容盘点
1. 对 scope 内每个 KP，提取以下要素：
   - 核心概念（一两句话）
   - 关键机理（如有）
   - 学习路径（前置 → 本节 → 后续）
   - 讲解脉络（来自 KP `source_notes` 关联提炼笔记的「📐 讲解脉络」段，如有）
   - 学生易错点（来自教学视角段）
   - 代表例题
2. 读取已提炼资料补全（v0.8 新增）：
   - 检查每个 KP 的 `source_notes` 和 `source_extracts` 字段，若关联了已提炼的教材/网课资料 → **必须读取**对应的提炼文件。
   - 提取可用资产：完整推导、数据表格、分类体系、教材原题、反例/口诀。
   - 若提炼文件中的内容比 KP 更充实 → 优先用提炼文件内容填充讲义，同时在讲义中标注来源（如"摘自 [[教学逻辑提炼-XX]]"）。
   - 若提炼文件中有 KP 未覆盖的推导/例题 → 补充进讲义的「完整推导折叠区」或课后作业。
3. 统计盘点：
   - 总知识点数
   - 关联的题型数
   - 关联的真题数
   - 缺口（教学视角段缺失？例题缺失？提炼资料是否已读取？）

#### 阶段 1.5：教学逻辑与深度边界确认

> ⭐ v0.7 强制：凡是新授课备课大纲或课堂讲义，都必须先完成本阶段；复习课/习题课可简化，但仍需确认学生已学边界。

1. **定位轮次与班型**：确认是第一轮/第二轮/第三轮/决赛，基础班/提高班/决赛班。
2. **读取教学逻辑来源**：优先级为 `[[模板-教学逻辑提炼]]` 产物 > 教师讲义/竞赛指导教程 > 教材章节 > 备课框架 > KP 教学视角。
3. **读取授课顺序总表（若涉及无机化学第一轮）**：若备课范围属于无机化学第一轮（原子结构→分子结构→晶体结构→配合物→气体溶液→热力学→平衡→酸碱→电化学→定量分析），先读取 `[[周坤无机新课-知识库自用授课顺序总表]]` 确认当前备课阶段在10阶段表中的位置和前后衔接。
4. **读取多源教学策略（若KP含多来源）**：若范围内 KP 的 `source_notes` / `source_extracts` 包含多个来源（如周坤+质心+教材），读取各来源的切入方式、反例、口诀，在备课大纲中标注可选策略，供教师按学情选择。
5. **确定讲授顺序**：不要按 KP 难度或考纲编号机械排列，而要从学生已有认知出发，写出“先讲什么 → 铺垫什么 → 何时形成结论 → 例题何时插入”。
4. **明确本轮边界**：填写 `depth_boundary` 与 `deferred_topics`，并说明课堂若被问到 deferred_topics 时如何一句话处理。
5. **基础班新授课默认结论后置**：课堂引入先给问题/现象，不在开头堆最终结论；核心结论放到课后总结或课堂后半段形成。
6. **缺失处理**：若没有现成教学逻辑来源，不得直接跳过；应在备课大纲“零、教学逻辑来源与深度边界”中标注“临时根据备课框架与 KP 教学视角提炼，待补正式 [[模板-教学逻辑提炼]] 产物”。

#### 阶段 2：教学结构设计（按 output_type 分化）

**若 output_type = 备课大纲（新授课）**：
1. 按 `[[模板-备课大纲]]` 组织：
   - **零、教学逻辑来源与深度边界**：来源表 + 本轮讲到哪里为止 + deferred_topics + 结论呈现策略
   - **一、教学定位**：课时安排表 + 可测量教学目标 + 教学重难点表
   - **二、知识脉络与授课路线**：路线总览（text 流程图）+ 认知台阶表（前置认知→新认知→讲解策略→深度标记→建议时长）+ 例题穿插策略表
   - **三、课时分配与内容编排**：精确到分钟的时间段表，每个时段标注引用素材（`[[专题-XX#section]]` 或 `[[KP-XX]]`）和媒介
   - **四、媒介分工表**：PPT vs 板书 vs 混合，附原因说明
   - **五、备课资源索引**：专题页 section 引用、KP 引用、真题引用
   - **六、课后追踪**：必做/选做作业
   - **七、教学反思**：时间控制/学生卡点/下一轮调整（课后填写）
2. **关键约束**：备课大纲**不包含知识内容本身**，只写"讲什么、怎么讲、用什么策略、讲多久"，知识内容通过 wikilink 精确引用专题页/KP

**若 output_type = 课堂讲义（新授课）**：
1. 学习目标以「建立」和「理解」为主（应用层 ≤ 30%）
2. 必须设计「课堂引入与本节问题」环节（生活现象/实验演示/认知冲突），基础班默认不在开头堆最终核心结论
3. 机理拆解必须分步、详细、带电子流说明
4. 例题以 ⭐~⭐⭐ 为主，目的为验证概念理解
5. 含完整知识内容（学生课后可独立阅读）
6. **内容补全策略（v0.8 强制）**：
   - 讲义的知识内容以 KP 为基底，但**不局限于 KP**。
   - 若 KP 的 `source_notes` / `source_extracts` 关联了已提炼的教材/网课资料 → **必须读取并整合**其中的推导、表格、例题、分类体系。
   - 若提炼资料中有完整推导而 KP 只有结论 → 在讲义「完整推导折叠区」中补充推导。
   - 若提炼资料中有教材原题而 KP 例题不足 → 优先选用教材原题作为课堂例题或课后作业。
   - 若提炼资料中有反例/口诀/记忆技巧 → 放入「易错点预警」或「课堂例题」的分析提示中。
   - 所有引用的提炼资料必须在讲义中标注来源 wikilink。
7. 必须写明本节 `depth_boundary` 与”暂不要求掌握”的 `deferred_topics`，防止学生误以为后续轮次内容本节就该掌握
8. “本节形成的核心结论”放在学生经历问题、推理和例题之后，以 2-4 条一句话总结呈现

**若 output_type = 课堂讲义（复习课）**：
1. 学习目标以「整合」「辨析」「提升」为主
2. 必须构建「知识网络大图」（覆盖 2-3 个关联 KP）
3. 设计「横向对比表」（易混点的并列比较）
4. 例题以 ⭐⭐~⭐⭐⭐ 为主，必须跨知识点综合
5. 归纳「方法策略」和「速判技巧」

**若 output_type = 课堂讲义（习题课）**：
1. 学习目标以「方法熟练」「速度提升」「准确率」为主
2. 以「题组」为单位组织（示范题 1 + 练习题 2-3 + 变式 1）
3. 每道题必须 explicit 提炼「识别信号」和「解题钥匙」
4. 题量：8-15 道，难度 ⭐⭐~⭐⭐⭐⭐
5. 预留「限时综合训练」环节（模拟考试压力）

**若 output_type = 专题页升级**：
1. 检查专题页各 section 完整性：
   - `## 一、核心结论汇总`：必须记住的结论 + 最高频决策路径 Mermaid 图
   - `## 二、对比表格`：≥3 行，含触发条件列
   - `## 三、解题套路 / 决策流程`：Step 检查清单 + 依据 KP wikilink
   - `## 四、反应机理拆解`（可选）：含检查表
   - `## 五、典型例题串讲`：≥2 道，覆盖"分析→解答→反思"三段式
   - `## 六、关联知识点` / `## 七、关联题型`
   - `## 八、相关真题`：Dataview 查询可用
2. 升级后 `status` 从 `框架` → `可用` 或 `初稿`

**课堂讲义共性设计**：
- **课时分配**（按 duration 和课型调整）：
  - 新授课：引入（8-15%）+ 核心讲解（50-60%）+ 例题（20-30%）+ 总结（10-15%）
  - 复习课：网络回顾（15-25%）+ 易错点攻坚（25-35%）+ 综合例题（30-40%）+ 总结（10-15%）
  - 习题课：目标说明（3-5%）+ 题组训练（70-80%）+ 限时训练（10-15%，可选）+ 快讲总结（5-10%）
- **难度梯度**：入门 → 巩固 → 提高 → 挑战（决赛级可选）

### 10.7 高级技巧

#### 技巧 1：跨 KP 主题贯穿
当 scope 是多个 KP 时，找出它们的「同源谱系」（如 Wittig / HWE / Lawesson 都属于 "P/S 抓 O" 驱动的 4 元环路径），用谱系图作为讲义主线。

#### 技巧 2：双视角讲义
- **物理视角**：化学是怎么发生的（机理）
- **教学视角**：学生该如何理解（类比/口诀）
- 两个视角并列，避免单纯堆砌机理但学生记不住

#### 技巧 3：习题驱动的备课
不从理论开始，而从一道关键题倒推：「这道题考什么？要解需要哪些前置 KP？」反向铺出讲解路径，更贴近实战。

#### 技巧 4：分层注释
讲义主线写「基础+提高」必讲内容，决赛拓展用 collapse / callout 标记，让基础学生跳过不影响主线，竞赛生展开。

### 10.8 边界与失败案例

**备课大纲特有的边界**：
- ❌ **备课大纲里粘贴了专题页正文**（备课大纲只能引用 wikilink，不能重复知识内容）
- ❌ 备课大纲缺少「教学逻辑来源与深度边界」（不知道依据什么顺序讲、讲到哪里为止）
- ❌ 备课大纲缺少「认知台阶表」（变成了简单的知识点罗列）
- ❌ 备课大纲缺少「媒介分工表」（没有说明 PPT/板书/混合的使用策略）
- ❌ 一个专题只产了一份备课大纲（基础/提高/决赛应有不同讲法）
- ❌ 不区分第一轮与第三轮，把同一 KP 的全部内容平铺进课件
- ❌ 参考多套网课资料时未做来源整合，导致同一知识点出现两套矛盾的讲授顺序或重复例题（必须遵循 `[[00-首页/方法论-多源教学资料整合]]`，统一映射到自己的认知台阶表）

**课堂讲义特有的边界**：
- ❌ 把全部 KP 内容简单平铺成讲义（没有筛选/取舍）
- ❌ 新授课基础班直接把专题页「核心结论汇总」搬到开头，跳过问题/现象/铺垫
- ❌ 例题只丢题面不写解答路径
- ❌ 学习目标写成「让学生学会 XX」（不可测量）
- ❌ 不区分听众层次，给基础生讲决赛级机理
- ❌ 基础班一开始讲完整量子数规则，把“轨道标签”提前讲成主线
- ❌ 第一轮氧化还原/电化学让复杂配平或高阶推导占用第一节课主线，挤掉原电池、E°、Nernst 的核心路径

**通用边界**：
- ❌ 修改源 KP（应只读）
- ❌ 不引用 wikilink（学生看完无法溯源）
- ❌ 专题页升级时添加了教学设计内容（如课时分配、媒介分工）——这些属于备课大纲

### 10.9 备课产出的 7 天复盘
> 实际授课后，可触发「备课复盘」分支：
> 1. 哪些段学生反应最好/最差
> 2. 哪些例题超时/卡住
> 3. 哪些易错点没料到
> 复盘信息可以追加到对应 KP 的「教学视角」段，形成正向反馈循环。

## Agent 11 · 答疑 Agent（占位 v0.5）

> 🚧 **当前状态**：占位轻量版。完整版待用户使用一段时间备课 Agent 后，根据真实问题模式再设计。

### 11.1 角色定位
- **何时使用**：用户提化学问题，希望基于本 KB 的内容回答（区别于通用 LLM 回答）
- **核心区别**：必须引用本 KB 的 wikilink，不只是凭知识回答

### 11.2 简化流程（v0.5）
1. **意图识别**：用户问的是
   - 概念解释 → 找最相关 KP，引用其「概念」段
   - 反应机理 → 找 KP 的「机理」段，可能涉及多 KP
   - 习题求解 → 找题目可能匹配的 KP 与题型，参考已有解答
   - 比较对照 → 找 2+ KP，对比关键属性
2. **回答模板**：
   ```
   ## <问题标签>
   
   <核心回答>
   
   ### 引用 KP
   - [[KP-1]]：<相关段落>
   - [[KP-2]]：<相关段落>
   
   ### 进阶阅读
   - [[相关专题]]
   - [[相关真题]]
   ```
3. **必做**：
   - 至少引用 1 个 KP（如确无相关 KP，明确告知"本 KB 暂无对应 KP，建议先用 Agent 2 填充"）
   - 不编造 KB 中不存在的内容
4. **不做**：
   - 不修改 KP
   - 不写到文件（除非用户要求"把这个回答存档"，则落到 `05-问答/`）

### 11.3 升级路线（v1.0 计划）
- 增加问题分类细化（概念/机理/合成路线/比较/真题/超纲拓展）
- 每类有专门 prompt 子模板
- 跨 KP 综合题的解题路径生成
- 支持苏格拉底式追问（不直接给答案，引导推导）

### 11.4 与其他 Agent 联动
- 答疑过程中发现 KP 内容空白 → Follow-up 提示用 Agent 2 填充
- 答疑被频繁问到的问题 → 月度复盘时考虑沉淀为新的 KP 或专题

## 总体使用建议

### 备课与答疑的边界
| 触发场景 | 用 Agent |
|---|---|
| 系统化产出讲义/大纲 | 10 备课 |
| 解答即时具体问题 | 11 答疑 |

```
KP 填充（Agent 2）
   ↓
KP 优化（Agent 7）
   ↓
备课产出（Agent 10）→ 实际授课 → 学生反馈
   ↓                                    ↓
复盘 → 教学视角段更新 → 下一轮备课更精准
```

### 不要让使用 Agent 污染源 KP
- 备课时发现"这个例题该补到 KP 里"→ **不要直接补**，挂到 Follow-up
- 由用户决定是否启 Agent 7 优化（保持单一职责）
