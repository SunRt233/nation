# Agent 提示词 — 化学竞赛知识库构建（构建型 1-6）

> 用于指导 Claude Agent 完成知识库**构建**的各个环节（创造新内容）。
> 每个提示词是独立可复用的，根据当前任务选用。

⚠️ **默认执行口径（v1.5.1）**：
- 日常任务先读 [[Agent最小执行协议]]
- 默认优先使用 `[[Agent-1-资料提炼-轻量]]` 到 `[[Agent-6-质量审查-轻量]]`
- 只有复杂、跨模块、批量、高风险或已失败任务，再补读 `[[Agent标准作业流程]]` 和本长版文件的对应章节

> **定位区别**：`[[Agent最小执行协议]]` 负责“默认读什么、哪里升级、收工回写哪里”；轻量提示词负责“这类任务的最小执行骨架”；本文件负责“复杂场景下的完整细则与特殊约束”。

> **新增衔接口径（2026-06-18）**：
> - 外部资料导入与资料提炼，不再只以“是否入库”为完成标准。
> - 新资料默认要为后续备课提供 6 类可抽料资产：教材主干、教学顺序、桥梁解释、例题链、易错点、图片/图景。
> - 后续继续导入的新书、新网课、新题库、新图包，默认纳入同一套抽料审计，而不是临时另开流程。

## 1. 资料提炼 Agent

**用途**：将教材章节提取为结构化提炼笔记，存入 `07-资料提炼/`

### 📚 模式选择（v1.3 起：执行前必须先决定）

> ⭐ v1.3 强制：动手前**先决策**用「每章模式」还是「总索引模式」。模板和提炼粒度不同，混用会导致下游 Agent (4/10) 误读。

| 维度 | 每章模式（默认） | 总索引模式 |
|---|---|---|
| 模板 | [[模板-资料提炼]] | [[模板-资料提炼-总索引模式]] |
| 文件粒度 | **一章一文件**，存 `07-资料提炼/{书名}/第X章-{标题}.md` | **一书一文件**，存 `06-外部资料导入/书籍/{书名}/{书名}-总索引.md` |
| 适用书 | 章节自包含的常规教科书（ABOC、《无机化学》、《有机化学》等）| ① 跨章主题主导 ② 单章体量小、合在一起更易理解 ③ 派生 KP 多于章节数 ④ 多作者论文集 |
| 标杆样本 | [[提炼-ABOC-第1章-绪论]] | [[Arrow Pushing in Inorganic Chemistry-总索引]] |
| 「📐 讲解脉络」粒度 | 单章作者讲解逻辑 | 全书核心思维 + 跨章主线 |
| 「考点抽象」位置 | §五 例题提取 | §七 关键例题与考点抽象（标杆例题，可缺失） |
| 「🪤 踩坑回流」 | 单章 4 问题 | 全书 4 问题 + 总索引特有的高频坑（跨章命名漂移等） |

#### 判断流程
```
源文件是哪种结构?
  ├── 常规教科书,每章独立成单元   → 每章模式 (默认)
  ├── 跨章思维主导(如 APiIC 把"think like a lone pair"贯穿 8 章) → 总索引模式
  ├── 论文集 / 综述 / 单元长度悬殊                    → 总索引模式
  └── 拿不准 / 边界案例 → 用户确认 (Phase 1)
```

> **不要混用**：① 已用每章模式做了 1-2 章后突然切换成总索引,会丢失已建索引;② 一本书选了总索引模式则**不**为该书在 07-资料提炼/ 下再单独建章节文件。

```
## 具体要求
1. **考纲映射**：对照 `[[中国化学奥林匹克基本要求-总览]]` 做边界校验，但不要机械只留“考纲明确写出”的内容。教材中稳定出现、且对后续课堂理解有价值的主干内容，可以保留，并在"超纲/非竞赛内容标注"或边界说明中写清理由。
2. **知识点对照**：列出本节/本书涉及的知识点，标注现有状态（骨架/初稿/已填充/缺失）。
3. **待建KP交叉比对（v1.2 新增）**：读取 `[[待建KP优先级清单]]`，筛选出与本次资料 subject/module 相关且状态为「待建」的 KP。在提炼笔记的「与待建KP优先级清单的对照」表格中：
   - 按频率从高到低列出所有匹配的待建 KP
   - 标注匹配程度（完全匹配 / 部分匹配 / 待验证）
   - 明确行动计划（本次新建 / 增量补充 / 暂不处理）
   - **频率 ≥5 的待建 KP 必须在本次提炼中处理（新建或补充），不得跳过**
3. **LaTeX 公式**：保持原格式，化学方程式用 `\mathrm{}` 包裹。
4. **图片处理**：原图为 `![](path)` 格式的，改为 wiki-link `![[mineru/path]]`。
5. **表格保留**：Markdown 表格格式保留。
6. **例题提取**：
   - 每章模式 → 统计并列出本节所有例题/习题的编号、对应的知识点和题型
   - 总索引模式 → 提取**标杆性**例题/反应/习题（不必每章都有）；纯机理叙述书无典型例题时明示跳过
7. **📐 讲解脉络（v1.1 强制）**：
   - 每章模式 → 在「二、📐 讲解脉络」用 3-5 句话提炼**作者本章的讲解逻辑**，至少回答以下子项中的 3 个：① 切入前置 ② 铺垫段落 ③ 全章高潮 ④ 一笔带过 ⑤ 教学风格
   - 总索引模式 → 在「二、📐 全书讲解脉络」回答：① 全书核心思维 ② 章节组织逻辑 ③ 跨章主线 ④ 教学风格 ⑤ 与传统教材的差异
   - **这一段是后续备课 Agent 和多源对照的最高价值锚点，不可省略**
8. **考点抽象（v1.1 强制）**：「例题提取」表格的"考点抽象"列必须填写**比题面更普遍的考点描述**。错误范例："求 $K_c$"。正确范例："$K_p \leftrightarrow K_c$ 量纲转换 + $\Delta n$ 符号易错"。这一列直接喂给题库拆分 Agent (Agent 4) 做 tag 聚合。
9. **🪤 踩坑回流（v1.1 强制 / v1.2 增补 `kp_target`）**：完成提炼后必须填写「🪤 踩坑回流」表格的 4 个问题：① 字符/公式/图片转换坑 ② 术语冲突 ③ mineru 漏图 ④ AI 困惑段落。**即使全部"无"也要明示**；任一问题答案非"无" → ① 在 Phase 4 同步追加到 [[经验沉淀]] 第八类「外部资料导入类」(8-x 编号递增)；② **v1.2 新增** 根据 `kp_target` 在对应 KP 的「十一、易错点」中插入 wikilink 引用。
   - **kp_target 规范**：若答案非"无"，必须在模板表格的 `kp_target` 列标注该坑点应回流到哪个 KP 的哪个段落，格式 `[[知识点名]]§段落编号/名`。示例：`[[配合物]]§十一`。涉及多 KP 时用逗号分隔。
   - 总索引模式特有的高频坑提示：跨章命名漂移 / 派生 KP wikilink 重名 / 章节号与 OCR 行号错位
10. **备课抽料视角（2026-06-18 新增）**：提炼时必须显式回答“这份资料未来如何服务备课”。至少盘点 6 类资产中的命中项：教材主干、教学顺序、桥梁解释、例题链、易错点、图片/图景。若某类为空，写明原因，不可默认跳过。

## 不要做的事
- 不要改动原书的公式和化学方程式
- 不要自行添加未在原书中出现的内容
- 不要跳过"看起来简单"的段落——竞赛可能考得很细
- **不要混用模式**：一本书一旦定了模式就不要中途切换；总索引模式下不要在 07-资料提炼/ 重复建该书章节文件
- **不要省略「📐 讲解脉络」**：这是 v1.1 的核心增量，不能填占位 `[待补充]`，必须真实提炼
- **不要把"考点抽象"填成题面复述**：见上述要求 8 的范例
- **不要把"踩坑回流"填空**：4 个问题必须有答案（"无"也算答案）；答案非"无"时必须填写 `kp_target`
- **不要遗漏 `source_notes` 回写**：Phase 4 必须把提炼笔记 wikilink 写入关联 KP 的 `source_notes`，建立双向追踪

## 输出格式
- frontmatter 必须含主版本字段 `version`；若模板仍保留兼容字段，再同步填写 `template_version` 且与 `version` 保持一致。其他必填字段：
  - 每章模式 → `source_type`、`subject`、`syllabus_codes`、`knowledge_points`
  - 总索引模式 → `type: 外部资料索引`、`template_mode: 总索引`、`source_book`、`subject`、`module`、`related_knowledge`
- 「📐 讲解脉络」段非空，至少 3 个子项有内容
- 「考纲映射」用表格列出每个考纲条目对应的内容
```

### 执行规则
1. **先保留原课顺序**：按原课出现顺序记录切入、铺垫、概念、例题、总结，不要一开始按 KP 重排。
2. **区分“原资料涉及”和“当前轮次展开”**：原课讲了不等于基础班必须讲；原课没讲但 KB 已有的高阶内容，也可能需要列入“本轮禁止提前展开”。
3. **例题记录教学作用**：例题不只摘题目，还要说明它用于引入、巩固、反例、综合还是升阶。
4. **两遍法处理 MD/PDF**：
   - 第 1 遍用 MD/转写抓讲课顺序、认知台阶、例题位置、轮次边界；
   - 第 2 遍在正式备课、讲义、PPT、KP 回流前，对照 minerU/PDF/图片复核复杂公式、表格数据、长题题面和图片。
5. **连续课程先单课后总版**：例如 L1/L2/L3 先分别提炼，再合并为“热力学初步-第一轮”总版；备课大纲优先引用总版作为 `logic_sources`。
6. **保持待审核状态**：产出默认 `status: 试点-待审核` 或 `试点-总版待审核`。
7. **文件标题错位检查（v1.5 新增 · 强制）**：网课OCR笔记/讲义常见文件名与正文严重不匹配（如 `10.化学平衡（一）` 实际讲热力学）。**必须以正文内容为准进行归类和映射**，不可凭文件名判断章节归属。在提炼文件顶部用 ⚠️ 醒目标注所有标题错位/内容混杂的文件。
8. **内容缺口自检（v1.5 新增）**：在 §2.5 或类似位置明确标注：
   - 哪些内容课堂笔记覆盖而讲义缺失；
   - 哪些内容仅讲义覆盖而笔记缺失；
   - 哪些内容两者都缺失（缺口）。
   缺口需标注补救建议（如"由教材补充"、"划归其他Batch"、"待后续资料补全"）。
9. **跨Batch缺口关闭追踪（v1.5 新增）**：若当前Batch发现此前Batch标注的缺口已被其他文件补全（如 Batch 3 标注MO理论缺失，Batch 4 发现 `04.晶体结构（一）` 实际含有MO理论），必须在当前文件中标注"缺口已关闭"，并回溯修正前序Batch文件的 §2.5 说明。
10. **OCR 噪声过滤（v1.5 新增）**：网课 OCR 笔记常含大量噪声，处理时必须过滤：
   - **必跳**：`<details><summary>text_image` 块内的手机截图碎片（电量、信号、无关符号）。该块通常不含有效教学内容。
   - **必跳**：连续 3 行以上仅含 `.` / `-` / 数字序列的空白填充行。
   - **保留参考**：时间戳（如 `L2 01:43:17`、`10.2 (Figures N)`）仅用于确认讲解顺序和原课时间结构，**不写入正文**。可在 §二「原资料讲解顺序」的「出现位置」列作为顺序标记。
   - **保留正文**：正文段落中的公式、概念描述、例题题干是核心提取目标。
   - **标记碎片化**：公式断裂、下标/希腊字母乱码处，标记 `⚠️ 据教材校正`，禁止自行编造公式。优先用已提炼的教材内容交叉校准。
11. **服务备课而不是只存顺序（2026-06-18 新增）**：网课提炼完成后，必须能回答：
   - 哪些内容可直接进入备课大纲的认知台阶或例题穿插
   - 哪些内容适合进入学生讲义的正文解释、易错提醒、自查句
   - 哪些内容只适合作为教师口令或课堂追问
   - 哪些图片、图景、演示动作值得进入 PPT 或讲义

## 批量处理策略

如果一次要处理很多文件（如填充 100+ 个知识点、拆分 200+ 道题），使用以下策略：

### 并行 Agent 模式

```

```

```
1. 抽查 3-5 个文件验证格式和内容
2. 检查 frontmatter 字段是否填写正确
3. 对于格式错误，写一个修正 Agent 批量修复
```

### 批量处理时的 SOP 要求

启动并行 Agent 之前，确保已完成 `[[Agent标准作业流程.md#Phase 0 — 任务理解与上下文加载]]` 和 `[[Agent标准作业流程.md#Phase 1 — 任务分析与计划制定]]`。

- 明确文件范围（如"处理 Ch02-Ch03 的知识点"）

### 常见踩坑
1. **路径**：Agent 使用 vault 内的相对路径（不用绝对路径）
2. **wiki-link**：格式为 `[[文件名]]` 不含 `.md` 扩展名

5. **题号**：保持原书编号体系，不要重新编号

## 1B. 资产回流 Agent（v1.0）

> **新增于 2026-05-31**。本 Agent 用于将 `[[模板-教学逻辑提炼]]` v1.1+ 中 §八「可提取资产清单」的高/中可信度资产，回流到对应的知识点(KP)文件中。
>
> **前置依赖**：必须先完成 [[1A.网课上课思路提炼]]，产出含 §八资产清单的教学逻辑提炼文件。

**用途**：打通"教学逻辑提炼 → 知识点知识库"的回流通道，使网课中的高价值数据、分类框架、认知冲突案例、反应机理等资产沉淀到可长期维护的 KP 中。

### 触发关键词
- "回流资产"、"把教学逻辑中的数据写进KP"
- "批量回流§八资产"、"继续回流"
- "提取资产到知识点"

### 总体工作流

```
教学逻辑提炼文件(07-资料提炼/教学逻辑提炼/)
    │
    ├─→ 读取 §八 资产清单 ──→ 按可信度排序 ──→ 检查目标KP是否存在
    │                                               │
    │                                    ├─ 存在 → 直接回流
    │                                    └─ 缺失 → 建立框架(模板-知识点 v1.3.2)
    │
    └─→ 逐项回流到目标KP ──→ 更新KP frontmatter(source_extracts)
            │
            └─→ 同步更新：源文件§八状态 + 备课待办§七总表 + 工作日志
```

### 前置检查清单（执行前必须完成）
- [ ] Phase 0：已读 `[[状态摘要]]`、`[[活跃任务]]`；必要时补读 `[[Agent战略方向]]`、`[[工作日志]]`、`[[备课与教学思路待办]]` §七
- [ ] Phase 0：已确认待回流的教学逻辑提炼文件路径，且文件已含 §八「可提取资产清单」
- [ ] Phase 0：已读 `[[11-模板/模板-教学逻辑提炼]]` v1.1+，理解 5 种资产类型和 Appendix A 回流格式模板
- [ ] Phase 1：已提取待回流资产列表（高/中可信度），并与用户确认本次回流范围
- [ ] Phase 1：已检查每个资产的目标KP是否存在。检查流程（强制）：
  1. `Grep` 搜索目标关键词（如"分裂能"、"Pauling规则"）
  2. 检查已有KP的 `aliases` 是否包含目标名
  3. `Read` 确认该KP的内容是否已覆盖此概念（可能作为子内容存在，如分裂能嵌于[[晶体场理论]]）
  4. **只有确实不存在任何覆盖时才新建KP框架**；否则将资产回流到已有KP的对应段落
  5. 禁止因"名字不同"就新建空壳KP（如周坤回流目标中的分裂能、Pauling规则、晶体缺陷、配合物命名，知识库中已有KP覆盖）
- [ ] Phase 1：已确认回流优先级：高可信度 > 中可信度；数据资产/分类资产 > 实例资产 > 反应资产/机理资产
- [ ] Phase 1：用户已确认或同意执行

### 执行后检查清单（执行后必须完成）
- [ ] Phase 3：抽查 3-5 个回流后的KP，确认：
  - 内容格式正确（使用 Appendix A 对应格式模板）
  - 来源标注完整（含资产ID、可信度、原始提炼文件wiki-link）
  - 未破坏KP原有结构（段落编号连贯、表格格式正确）
- [ ] Phase 3：检查目标KP的 frontmatter `source_extracts` 字段是否已更新（新增对应资产条目）
- [ ] Phase 3：检查KP的 `updated` 字段是否已更新为当前日期
- [ ] Phase 3：更新教学逻辑提炼源文件 §八 中的资产「回流状态」列（⏳ 待回流 → ✅ 已回流）
- [ ] Phase 3：更新 `[[备课与教学思路待办]]` §七 资产总表中的对应状态
- [ ] Phase 3：更新 §七 回流统计数字
- [ ] Phase 4：在 `[[工作日志]]` 记录：
  - 本次回流的资产数量、来源文件、目标KP列表
  - 新建KP框架的数量和名称
  - 遇到的问题（如目标段落缺失、格式冲突等）

### 资产类型与回流目标映射表

| 资产类型 | 推荐目标KP段落 | 回流格式 | 典型内容 |
|:---|:---|:---|:---|
| **数据资产** | §五（常见分类/情形）或 §十二（教学视角） | 数据表格 + 来源标注 + 可信度说明 | 键长、键能、熔点、相变温度等 |
| **分类资产** | §四（关键结论）或 §五（分类表格） | 对比表格 + 定义/规则说明 | 结构语法、对比框架、分类体系 |
| **实例资产** | §七（易混点）或 §十二（教学视角） | 案例描述 + 认知冲突分析 + 数据 | 反常现象、打破直觉的结构实例 |
| **反应资产** | §四（关键结论/机理） | 反应序列/连续谱 + 结构-性质关联 | 活化连续谱、反应路径 |
| **机理资产** | §三（核心原理）或 §四（机理分析） | 机理图描述/能级图 + 分步解释 | MO图像、轨道相互作用、双重机制 |

### 可信度处理规则

| 可信度 | 处理方式 | 标注要求 |
|:---|:---|:---|
| **高** | 直接回流，作为KP正式内容 | 标注来源文件和资产ID |
| **中** | 回流但附加验证提示 | 必须标注"原始笔记为课堂口述，建议查文献确认" |
| **低** | **不回流**，仅在备注中记录原因 | — |
| **暂不回流** | 保持 ⛔ 状态，等待后续条件成熟 | 记录阻塞原因和目标轮次 |

```markdown
|:---|:---:|:---|
```

### 缺失KP处理流程

若资产的目标KP不存在：
1. 使用 `[[11-模板/模板-知识点]]` v1.3.2 创建框架文件
2. frontmatter 中 `status: 框架`、`teaching_ready: false`
3. 在 §四~§七 中填入该资产的内容（作为首个填充内容）
4. 在 `[[备课与教学思路待办]]` §三「专题页升级待办」中登记该KP（如尚未登记）

### 常见踩坑

1. **目标段落不存在**：某些KP缺少§十二（教学视角）或§七（易混点），回流前需先检查目标段落是否存在，不存在则在合适位置新建子段落
2. **三段同步负担**：回流后必须同步更新 ①源文件§八 ②备课待办§七 ③KP frontmatter，漏任何一处都会导致追踪断裂
3. **格式翻译不是复制粘贴**：资产在教学逻辑提炼中是"清单式"的，回流到KP时需要按目标段落语境改写为"知识式"或"教学式"
4. **中可信度资产不要直接当事实用**：必须标注待验证提示，避免学生记住错误数据
5. **批量替换时的统计表陷阱**：更新源文件状态时，注意不要把备课待办§七的统计表也一并替换掉

### 与其他Agent的协作边界

- **与 Agent 1A 的关系**：Agent 1A 产出教学逻辑提炼文件（含§八），Agent 1B 消费该产出。1A 不负责回流，1B 不修改教学逻辑。
- **与 Agent 2 的关系**：Agent 2（知识点填充）从教材/讲义填充KP的系统性内容；Agent 1B 从网课提炼中补充精选资产。两者内容可共存于同一KP的不同段落。
- **与 Agent 7 的关系**：若回流后发现KP内容质量不佳，转交 Agent 7（KP优化）处理，而非在回流阶段大修。
