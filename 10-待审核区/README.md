# 待审核区（Pending Review）

## 作用

待审核区是知识库的"缓冲区"。所有**新增或修改的内容**在进入正式区域前，先在此停留，经确认后再迁移。

## 工作流程

```
外部资料 / AI生成内容 / 用户草稿
    ↓
06-外部资料导入 / 本待审核区
    ↓
提炼、整理、格式化
    ↓
07-资料提炼（中间态）
    ↓
审核确认（准确性、完整性、格式规范）
    ↓
03-知识点 / 04-专题与题型 / 04-题库（正式区）
```

## 文件命名规范

- `待审-知识点-[名称]-[日期].md`
- `待审-题目-[编号]-[日期].md`
- `待审-专题-[名称]-[日期].md`

## AI 产出分级规则（P0/P1/P2）

> 根据修改范围和影响程度，决定 Agent 产出直接进入正式区，还是先进入待审核区。

| 级别 | 判定标准 | 处理流程 | 示例 |
|:---|:---|:---|:---|
| **P0** | 修改 ≤3 个文件，内容级小修，无结构性变更 | **直接进正式区**，Phase 4 在工作日志记录即可 | 修正1个错别字、补1个wikilink、给1个知识点加Dataview |
| **P1** | 批量修改/新建内容，影响 4-20 个文件，或涉及新内容填充 | **先进待审核区**，用户确认后迁移到正式区 | 批量填充10个知识点、新建5个专题页、为50个文件加字段 |
| **P2** | 结构性变更：改模板、改SOP、批量重命名、删目录、改frontmatter规范 | **必须用户确认后执行**，执行前在Phase 1计划书中明确变更范围和回滚方案 | 重命名考纲条目文件、修改模板v1.1→v1.2、删除目录、改subject枚举值 |

### 执行细则
1. **P0 级**：Agent 执行后直接归档，无需用户逐条确认。但须在 Phase 4 汇报中明确列出修改了哪些文件。
2. **P1 级**：Agent 产出先写到 `10-待审核区/`，文件命名遵循 `待审-{类型}-{名称}-{日期}.md`。用户审阅后由 Agent 迁移到目标目录，并在工作日志记录迁移动作。
3. **P2 级**：Agent 在 Phase 1 必须起草详细变更计划（含回滚方案），经用户明确同意（如"可以执行""放手做"）后才能执行。执行过程中如遇意外，立即暂停并汇报用户。
4. **模糊地带**：若 Agent 不确定属于 P0 还是 P1，默认按 P1 处理（进待审核区）。宁保守，不冒进。

## 审核检查清单

- [ ] 内容准确性：化学事实、公式、数据是否正确
- [ ] 考纲对应性：是否对应正确的考纲条目
- [ ] 格式规范性：是否符合知识点/题目/专题模板
- [ ] 链接完整性：Wiki-links 是否指向存在的文件
- [ ] 来源标注：是否注明资料来源
- [ ] **分级合规性**：本次产出是否按 P0/P1/P2 规则落在正确的区域
