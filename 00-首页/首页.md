  .filter(f => f.path.startsWith(logFolder + "/"))
  .sort((a, b) => b.stat.mtime - a.stat.mtime);

for (const file of logFiles) {

## 🗺️ 知识库架构全景图

> 当前全景图文件：[[知识库架构全景图.canvas]]
> 下面这版是文字真相源，方便快速判断“信息从哪里来、加工后流向哪里、最后如何回流闭环”。

```text
外部资料 / 网课 / 教材 / 真题
    ↓ 导入
06-外部资料导入
    ↓ 提炼
07-资料提炼
    ├─ 章节/书籍提炼：提知识、例题、图表、考纲映射
    └─ 教学逻辑提炼：提认知台阶、轮次边界、可回流资产
            ↓ 双向回流
03-知识点（定义、原理、易错点、教学视角）
    ├─→ 04-专题与题型/专题：知识模块 + 解题引擎
    ├─→ 04-专题与题型/题型：方法/题型聚合
    └─→ 04-题库：通过 knowledge_points 与专题/备课联动

备课思路 / 多源整合方法论 / 教学逻辑提炼
    + 知识点 / 专题页 / 题库
    ↓
04-课件/备课大纲（教师教学设计）
    ↓
04-课件/新授课（学生课堂材料）
    ↓
学生侧材料 / 课后反馈 / 复习与习题课扩展

执行与治理闭环
活跃任务（短期执行真相源）
    + 工作日志（过程与证据）
    + 状态摘要（AI 会话快照）
    + Agent战略方向（长期优先级）
    + SOP / Agent 提示词 / 数据库
```

**一句话理解各层职责**：
- 考纲层负责“边界与覆盖”，不承载正文主内容。
- 提炼层负责“把外部资料转成可消费资产”，尤其补上“怎么讲”的教学逻辑。
- KP / 专题 / 题库层负责“知识组织、解题组织、检索组织”。
- 备课层负责“教师怎么讲”，讲义层负责“学生实际看到什么”。
- 任务 / 日志 / 摘要 / 战略层负责“下一步做什么、做过什么、为什么这么做”。
