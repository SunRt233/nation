# Agent操作与知识库导航说明

> 本文件解释 Claudian（AI Agent）在知识库中的操作方式，帮助你理解文件结构、跟踪进度、自主导航。

---

## 一、Agent 做了什么？（操作速览）

当你发出指令后，Agent 会按照 **Phase 0-4 标准作业流程** 执行：

```
Phase 0: 加载上下文（默认读 [[状态摘要]] + [[Agent战略方向]] + [[活跃任务]]，跨模块大批量任务时读完整模式）
    ↓
Phase 1: 制定计划（确认目标、选定工具、预判风险）
    ↓
Phase 2: 执行操作（批量创建/修改文件、提取资料）
    ↓
Phase 3: 验证抽查（检查 3-5 个产出文件的质量）
    ↓
Phase 4: 归档收尾（记录工作日志、更新活跃任务、更新索引、同步状态摘要/战略方向）
```

**你可以在以下文件中查看每次操作的详细记录：**
- `00-首页/工作日志/YYYY-MM-DD.md` —— 每次对话结束后 Agent 会在当天日志文件追加条目
- `00-首页/活跃任务.md` —— 任务卡聚合总览页
- `00-首页/状态摘要.md` —— AI 启动快照、风险与关键状态
- `00-首页/Agent战略方向.md` —— 当前阶段判断、长期优先级与默认推进方向

## 三、Dataview 查询是什么？（动态表格）

你在很多页面底部看到的「关联真题」表格是 **Dataview 插件**自动生成的，不需要手动维护。

### 它的工作原理

```dataview
TABLE year, exam_stage, difficulty
FROM "04-题库/真题"
WHERE contains(knowledge_points, this.file.name)
```

**白话解释**：
- 扫描 `04-题库/真题/` 文件夹下的所有题目
- 检查每道题的 `knowledge_points`（知识点标签）
- 如果标签中包含「当前页面名称」，就把这道题列出来

### 你看到的效果

| 题目 | 年份 | 题型 | 难度 |
|------|:---:|:---|:---:|
| 题-035-7-过渡金属配合物推断 | 2021 | 推断题 | 5 |

> ✨ **这意味着**：每当你新建一道真题并正确填写 `knowledge_points`，所有相关 KP 页面会自动「长」出这条记录。

### 如何手动刷新

如果新增题目后表格没有更新：
1. 按 `Ctrl + P` 打开命令面板
2. 输入 `Dataview: Force refresh all views`
3. 回车

## 五、Frontmatter 字段速查

每个文件顶部的 YAML 块（两个 `---` 之间）是文件的「身份证」。

### 真题文件常用字段

| 字段 | 示例值 | 用途 |
|------|--------|------|
| `type` | 题目 | 区分题目/知识点/索引 |
| `year` | 2021 | 按届筛选 |
| `module` | 无机和结构化学 | 按模块筛选 |
| `question_type` | 推断题 | 按题型筛选 |
| `difficulty` | 1~5 | 按难度筛选 |
| `teaching_level` | 入门/巩固/拓展/竞赛拔高 | 按教学层次筛选 |
| `knowledge_points` | [配合物, 晶体场理论] | **Dataview 关联查询的关键** |
| `tags` | [化竞, 真题, 35届] | 快速标签检索 |

### 知识点文件常用字段

| 字段 | 示例值 | 用途 |
|------|--------|------|
| `type` | 知识点 | 区分类型 |
| `status` | 骨架/初稿/已填充 | 填充进度跟踪 |
| `syllabus_code` | [12] | 对应考纲条目编号（单条单个值，多条用数组）|
| `importance` | 1~5 | 知识点重要程度 |

## 六、常见问题（FAQ）

**Q1: Agent 修改了文件，但我没看到变化？**
> A: Obsidian 有时不会实时刷新外部修改。按 `Ctrl + R` 刷新当前页面，或切换一下笔记再切回来。

**Q2: Dataview 表格显示「0 条结果」？**
> A: 检查两点：(1) Dataview 插件是否已启用；(2) 当前页面的文件名是否精确匹配题目 `knowledge_points` 中的值。注意大小写和全角半角。

**Q3: 我想自己添加一道真题，应该复制哪个模板？**
> A: 复制 `11-模板/模板-真题导入检查清单.md` 中的完整示例，或参照任意已有的 `题-035-xxx.md` 文件。

**Q4: Agent 提到的 "follow-up" 是什么意思？**
> A: 这是 Agent 在本次操作完成后建议的「下一步行动」。现在短期 follow-up 默认落到 `00-首页/活跃任务/` 任务卡，并由 `[[活跃任务]]` 聚合展示；工作日志只保留过程与证据；这些建议不会自动执行，除非你下次明确授权，或你说“继续推进”让 Agent 按优先级自主处理。

**Q5: "🔴 未建 / 🟡 初稿 / ✅ 已建" 这些标记什么意思？**
> A: 这是题目/知识点的工作状态图例：
> - 🔴 未建：文件尚未创建
> - 🟡 初稿：文件已创建但内容不完整
> - ✅ 已建：文件已完成并验证

## 七、如何与 Agent 高效协作

### ✅ 高效指令示例

| 低效指令 ❌ | 高效指令 ✅ | 原因 |
|-----------|-----------|------|
| "帮我整理一下真题" | "将 35届初赛第3题拆分为独立文件，按模板格式" | 明确届号、题号、格式要求 |
| "检查有没有错误" | "抽查题-035-7的frontmatter完整性，重点检查knowledge_points是否有broken link" | 明确范围、明确检查项 |
| "继续" | "继续创建35届剩余未建题目，从题-035-3-2开始" | 明确起点和终点 |

### 🚫 Agent 的红线（不会做的事）

1. **不会整体重写已填充 KP**：如果文件已存在且 status = "已填充"，Agent 默认只追加/修正，不整体重写。例外：明确触发的 Agent 7（KP 优化）或模板迁移任务，可在保留内容的前提下升级 frontmatter/段落结构
2. **不会删除文件**：如需清理，会建议你先确认
3. **不会修改 `.obsidian/` 配置文件夹**：除非明确授权
