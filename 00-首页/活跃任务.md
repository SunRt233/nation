# 活跃任务 · 结构化任务总览

> **用途**：本页是活跃任务系统的总入口，不再手写维护整份任务清单。
> **分工**：
> - `00-首页/活跃任务/`：每个任务一张独立任务卡，是短期执行真相源
> - `[[活跃任务]]`：聚合总览、筛选入口、使用规则
> - `[[工作日志]]`：记录过程、依据、验证结果、证据链接
> - `[[状态摘要]]`：读取任务卡聚合结果，生成 AI 启动快照
> - `[[Agent战略方向]]`：只保留长期方向，不再承载短周期任务清单

## 使用规则

1. 新任务优先新建一张任务卡到 `00-首页/活跃任务/`
2. 每张任务卡必须维护 frontmatter，至少包含：`status`、`priority`、`area`、`source_notes`、`related_notes`、`evidence`
3. 任务完成后，不删除卡片；改为 `status: completed` 并补 `completed`
4. 任务卡记录“还有什么没做”；任务经过与验证细节写入 `[[工作日志]]`
5. `[[状态摘要]]` 和数据库看板只从任务卡聚合，不再从日志正文扫描短期 follow-up

## 任务面板

### 🔴 P1 高优

```dataview
TABLE WITHOUT ID
  file.link as 任务,
  area as 领域,
  join(related_notes, " · ") as 关联,
  join(evidence, " · ") as 证据,
  updated as 更新
FROM "00-首页/活跃任务"
WHERE type = "活跃任务卡"
  AND status = "active"
  AND priority = "P1"
SORT file.name ASC
```
