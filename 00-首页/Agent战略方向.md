> **用途**：供用户和 Agent 快速理解“现在该做什么、不该做什么”。
> **维护方式**：用户主导调整长期方向；Agent 在阶段状态、备课进度、待决策事项发生明确变化时，应于 Phase 4 同步本文，并在 [[工作日志]] 中记录变更依据。

## 二、工作大方向（按优先级降序）

### 🔴 方向 A：内容深化（最高优先级）

**目标**：让现有 KP 从“有”变成“好用”，能直接支撑备课和学生自学。

**具体动作**：
- **教学视角补全**：每个 KP 的 §12（教学视角）是否完整？尤其关注：
  - 学生常见认知误区（附原因分析 + 正确认知 + 口诀/方法）
  - 入门级例题（附预期解答路径 + 教师引导提问）
  - 与现实/直觉的连接（类比、反例）
- **例题质量提升**：现有例题是否覆盖典型考法？是否附“分析→解答→反思”三段式？
- **易错点精细化**：不只是罗列错误，要解释“学生为什么会这么想”。
- **竞赛拓展**：天然产物全合成案例、近年新反应、不对称催化等前沿应用。

**触发条件**：Agent 接到“备课某专题”任务时，自动检查相关 KP 的教学视角完整性，缺则挂 Follow-up 或先补。

### 🟠 方向 B：教学逻辑提炼与轮次深度切分（高优先级）

**目标**：建立“教学路径数据库”，解决 KP → 专题 → 备课 → 讲义过程中“知识怎么排、讲到哪、哪些暂不讲”的问题。

**具体动作**：
- 从教师讲义、竞赛指导教程、教材章节中提取“作者/教师怎么讲”的逻辑：先讲什么、后讲什么、哪里铺垫、何时形成结论。
- 为同一主题建立第一轮/第二轮/第三轮的深度边界：本轮目标、本轮讲到哪里为止、本轮禁止提前展开。
- 备课前必须检查“本轮讲到哪里为止”和“本轮禁止提前展开”，缺失时优先补边界表。
- 新授课基础班不默认“核心结论先行”，应采用“问题/现象 → 铺垫 → 推理 → 形成结论”的路径；复习课/习题课可按需要结论先行。
- 将后续导入资料同时落到 [[模板-资料提炼]] 的“教学逻辑锚点”和 [[模板-教学逻辑提炼]]，不再只提知识点。

**触发条件**：
- 用户导入竞赛指导教程/教师讲义/教材章节；
- 用户要求生成新授课备课大纲或学生讲义；
- 已暴露出“基础班提前讲高阶内容”“复杂专项挤占主线”等问题。

### 🟠 方向 C：课件与教学产出（高优先级）

**目标**：把 KB 知识转化为可直接上课的材料。

**三层主线**：
1. **专题页**：知识模块 + 解题引擎，供学生自习、复习课/习题课投影。
2. **备课大纲**：教师教学设计，解决认知台阶、课时分配、媒介分工、深度边界。
3. **学生讲义**：学生可见的课堂材料，紧贴实际讲授内容。

> 课件大纲/PPT 大纲降级为 **精品课或正式 PPT 前的可选层**，常规新授课不强制经过“课件大纲”中间层。

**具体动作**：
- **专题页从“可用”升级为“精品”**：专题页是“知识模块+解题引擎”，需补充核心结论、对比表格、决策路径、解题套路、典型例题、相关真题 Dataview。
- **备课大纲生成**：使用 [[模板-备课大纲]]，从专题页、KP、教学逻辑边界出发产出教师教学设计。
- **课堂讲义生成**：使用 [[模板-新授课]]，产出学生可见的课堂材料；基础班新授课默认结论后置。
- **方法微专题落地**：把 [[模板-方法微专题]] 中的 15-20 分钟思维工具课变成具体课件。
- **习题课/复习课**：按真题策展模式填充，围绕核心方法和高频易错点组织。

**备课产出标准**（什么算“完成”）：
| 产出类型 | 完成 Checklist |
|:---|:---|
| 备课大纲 | frontmatter 完整 + 教学逻辑来源与深度边界 + 认知台阶表（≥4 个台阶）+ 课时分配 + 媒介分工表 + 例题穿插策略 + 备课资源索引 + 教学反思占位 |
| 课堂讲义 | frontmatter 完整 + 课堂引入/本节问题 + 核心内容 ≥3 个知识点 + 课堂例题 ≥3 道 + 本节形成的核心结论 + 易错点预警 ≥2 条 + 下节衔接说明 |
| 专题页 | 状态从 `框架` → `可用`：核心结论 + 决策路径 Mermaid + 对比表格 ≥3 行 + 解题套路含检查清单 + 典型例题 ≥2 道 + Dataview 查询 |
| 方法微专题 | 核心思想（1 句话）+ 适用场景表 + 操作步骤（流程图/口诀）+ 典例演示（1 道）+ 即时演练（1 道）+ 迁移口诀 |
| 习题课 | 主题聚焦（1-2 个方法）+ 真题 3-5 道（⭐→⭐⭐⭐ 递进）+ 常见错误汇总 |

**触发条件**：用户指定“备某课”或“做某专题课件”时执行；无明确任务时，按 [[备课思路/上课顺序（详细内容参考2024考纲）.md]] 的进度自动推进下一个未完成的课件，但先做教学逻辑/轮次边界检查。

### 🟡 方向 D：图片资产持续挖掘（中优先级 · 按需）

**目标**：让已有 mineru 图片被充分利用，而非预先扫描所有图片。

**具体动作**：
- **备课驱动**：备课某 KP 时，按需搜索 mineru 中是否有可用图。
- **高优待补清零**：Dataview 自动列出 `images_priority = high` 且 `has_images != true` 的 KP，Agent 按此清单补图。
- **真题图归档**：每处理一道带图真题，判断是否需要建图注笔记或嵌入相关 KP。

**禁止动作**：
- ❌ 不预先批量扫描所有 mineru 目录（ROI 极低，且大量图为 OCR 噪声）。
- ❌ 不为一次性用图建立图注笔记（需满足 ≥2 KP 引用或 ≥2 真题引用条件）。

### 🟢 方向 E：题库与 KP 联动（中优先级 · 后台）

**目标**：提升 knowledge_points 字段的 wikilink 覆盖率，让真题和知识点双向可导航。

**具体动作**：
- 对 plain text 标签，若对应 KP 已存在 → 转为 wikilink。
- 对 `KP-` 前缀旧格式标签 → 清理前缀。
- 对描述性标签 → 评估是否需要建独立 KP，不需要则保留 plain text。

**触发条件**：用户明确说“修复题库链接”或“审计 knowledge_points”时执行；不作为日常默认任务。

### 🔵 方向 F：复习周期与过程分训练（低优先级 → 中优先级 · 模板已就绪）

**目标**：利用 KB 的 `review_cycle` 字段和过程分模板，建立学生侧的学习闭环。

**体系框架**：
- **记忆层**：KP 闪卡（概念/公式/方程式/对比/信号卡，A4 裁切 8 张/页）→ `[[11-模板/模板-学生闪卡]]`
- **理解层**：专题思维导图、易错点便携版 → 从专题页核心结论+对比表提取
- **应用层**：专题练习卷（分轮次配比）、方法策略卡（A6 口袋卡）→ `[[11-模板/模板-专题练习卷]]` + `[[11-模板/模板-方法策略卡]]`
- **元认知层**：轮次自查清单、错题归因模板、诊断报告 → 从新授课自查清单+过程分评分表汇总

**具体动作**：
- 按轮次配套学生侧材料：第一轮以闪卡+基础练习卷为主；第二轮新增性质速记卡+过程分训练；第三轮达到完整四层（含机理 Image Occlusion 闪卡、逆合成策略卡、诊断报告）。
- 过程分评分表在真实课堂验证后迭代。
- 设计基于 review_cycle 的间隔复习清单（与闪卡 Leitner 盒子法联动）。

**触发条件**：
- 用户明确说”生成学生侧材料”或”做闪卡/练习卷/策略卡”时执行。
- 备课某专题时，可顺手产出配套闪卡 5-10 张（从当课 KP 提取）。

## 三、Agent 默认行为（当用户未明确任务时）

如果用户只说“继续推进”或“开始工作”而没有具体指令，Agent 按以下顺序自主决策：

```text
1. 检查 [[活跃任务]] 中的 🔴 P1 高优
   → 有未完成项 → 优先执行

2. 检查下一课件/备课任务前置条件
   → 目标专题若缺少教学逻辑来源或轮次深度边界 → 先补 [[模板-教学逻辑提炼]] 或边界表
   → 明确本轮讲到哪里为止、哪些 deferred_topics 禁止提前展开

3. 检查 Dataview「高优待补」图片清单
   → 有 high priority 缺图 KP → 按需补图（方向 D）

4. 检查 [[备课思路/上课顺序（详细内容参考2024考纲）.md]] 当前进度
   → 下一课件的备课大纲/学生讲义尚未生成 → 按模板产出（方向 C）
   → 常规新授课不强制先生成课件大纲；精品课/PPT 前再使用课件大纲

5. 随机抽查 3-5 个核心 KP（importance ≥ 4 且 updated 较早）
   → 教学视角不完整 → 补全（方向 A）

6. 如无上述触发 → 向用户汇报当前状态并请示具体方向
```

### 6.3 分轮次产出要求

同一专题在不同轮次中出现时，原则上按该轮次定位产出对应深度。若同一专题在多个轮次中内容差异显著，则需产出多个版本：

| 深度定位 | 课件差异 | 例题差异 | 作业差异 |
|:---|:---|:---|:---|
| **入门深度（通常对应第一轮）** | 重点放概念理解、直觉建立和基础工具；跳过高阶理论完整性 | ⭐⭐ 为主，少量 ⭐⭐⭐ | 以教材习题 + 初赛 ⭐⭐⭐ 以下真题为主 |
| **深化深度（通常对应后续轮次）** | 增加推导过程、横向联系、前沿应用和竞赛拓展 | ⭐⭐⭐ 为主，挑战 ⭐⭐⭐⭐ | 以初赛 ⭐⭐⭐⭐ + 决赛基础题为主 |
| **决赛/拓展深度** | 追求理论完整性与高阶模型 | ⭐⭐⭐⭐-⭐⭐⭐⭐⭐ | 决赛题、专题论文/教材习题 |

> 具体每一轮次涉及该专题的哪些内容，由轮次规划和教学逻辑边界表决定。第一轮尤其要防止把第三轮深化内容平铺进新课。

### 6.4 备课驱动的工作流

Agent 执行备课任务时，按以下流程推进：

```text
Step 1: 读取备课框架
  ↓ 读取 [[备课思路/第一轮总体备课框架]] 或 [[第二轮总体备课框架]]
  ↓ 定位目标专题的核心内容、课堂例题、轮次定位

Step 1.5: 确认教学逻辑与深度边界
  ↓ 检查是否已有 [[模板-教学逻辑提炼]] 产物、教师讲义/教材讲解脉络、轮次边界表
  ↓ 明确：本轮讲到哪里为止 / 本轮禁止提前展开 / deferred_topics
  ↓ 若缺失 → 先补边界表，再进入 Step 2

Step 2: 检查相关 KP 深度
  ↓ 按该专题涉及的 KP 列表，逐个检查 §12 教学视角完整性
  ↓ 若缺失 → 先补 KP 或挂 Follow-up，再进入 Step 3

Step 3: 生成备课大纲
  ↓ 使用 [[模板-备课大纲]]，产出不同轮次的版本（如需要）
  ↓ 备课大纲必须包含“教学逻辑来源与深度边界”

Step 4: 生成学生讲义
  ↓ 使用 [[模板-新授课]]，填充核心内容、例题、易错点、衔接
  ↓ 基础班新授课默认“问题/现象 → 铺垫 → 推理 → 结论后置”

Step 5: 配套产出
  ↓ 课后作业推荐（从题库筛选 3-5 道 wikilink）
  ↓ 如需图片 → 走图片检索工作流（[[图片索引]] + mineru 搜索）
  ↓ 精品课/PPT 制作前，可选生成课件大纲

Step 6: 复盘记录
  ↓ 在 [[04-课件/复盘索引]] 中登记本课件的复盘要点
  ↓ 课后根据实际教学反馈更新 → 移入 [[04-课件/已复盘/]]
```
