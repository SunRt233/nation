## 一、整理结论

### 1.1 是否需要重新建文件夹？

**不需要新建新的顶层文件夹。**  
当前 PDF 位于：

```text
07-资料提炼/原始资料/网课/
```

这个位置是合适的：它保留了“原始资料”的性质，又归属于 `07-资料提炼` 体系，后续可以自然衔接 [[模板-资料提炼]] 与 [[模板-教学逻辑提炼]]。

### 1.2 后续是否需要转换成 Markdown？

**需要，但不是把 159 个 PDF 一次性全部机械转换成 Markdown。**

更合理的做法是分三层：

| 层级 | 产物 | 存放位置 | 是否必须 |
|:---|:---|:---|:---:|
| 原始层 | PDF 原件 | `07-资料提炼/原始资料/网课/` | ✅ 已有 |
| 内容层 | 资料提炼 Markdown | `07-资料提炼/网课资料/<课程名>/` | 按需 |
| 教学层 | 教学逻辑提炼 Markdown | `07-资料提炼/教学逻辑提炼/<课程名>/` | 备课优先主题必须 |

**判断规则**：

| PDF 类型 | 后续动作 | 模板 |
|:---|:---|:---|
| 主要是知识点、公式、例题、图表 | 转成资料提炼笔记 | [[模板-资料提炼]] |
| 明显体现老师怎么讲、例题何时出现、先讲后讲顺序 | 转成教学逻辑提炼笔记 | [[模板-教学逻辑提炼]] |
| 既有知识内容又有讲课顺序 | 两份都生成：资料提炼 + 教学逻辑提炼 | 两个模板都用 |
| 当前不服务备课 | 暂不转换，只保留索引 | — |

## 四、后续 Markdown 转换策略

### 4.1 推荐转换顺序

```text
1. 先转“教学逻辑提炼”
   目标：解决怎么讲、先讲什么、讲到哪里为止

2. 再转“资料提炼”
   目标：补知识点、例题、图表、踩坑

3. 最后接入备课大纲
   在 frontmatter 的 logic_sources 中引用教学逻辑提炼笔记
```

### 4.2 不推荐的做法

- 不建议一次性把 159 个 PDF 全部转成 Markdown。
- 不建议把 PDF OCR 全文直接塞进知识库。
- 不建议把网课提炼结果直接写进 KP，除非经过 [[模板-资料提炼]] 的并入计划。
- 不建议把原始 PDF 移到 `04-课件/`；它们不是正式课件产物。

### 4.3 单份 PDF 转换后的命名建议

资料提炼：

```text
07-资料提炼/网课资料/<课程名>/资料提炼-网课-<课程名>-<主题>.md
```

教学逻辑提炼：

```text
07-资料提炼/教学逻辑提炼/<课程名>/教学逻辑提炼-网课-<课程名>-<主题>-第一轮.md
```

例如：

