## 一、按届次汇总

```dataview
TABLE WITHOUT ID

  year as 年份,
  module as 模块,
  question_type as 题型,

  teaching_level as 层次,

```

## 二、按知识点交叉筛选

> 低频参考视图。用于观察高频考点交叉覆盖；真要组卷时，优先回到页首交互面板。

```dataviewjs
const pages = dv.pages('"04-题库/真题"').where(p => p.type == "题目").array();
const counter = new Map();

for (const page of pages) {
  const points = Array.isArray(page.knowledge_points)
    ? page.knowledge_points
    : (page.knowledge_points ? [page.knowledge_points] : []);
  for (const point of points) {
    const key = String(point ?? "").trim();
    if (!key) continue;
    counter.set(key, (counter.get(key) ?? 0) + 1);
  }
}

const rows = [...counter.entries()]
  .sort((a, b) => b[1] - a[1] || a[0].localeCompare(b[0], "zh-Hans-CN"))
  .slice(0, 30)
  .map(([point, count]) => [point, count]);

if (!rows.length) {
  dv.paragraph("未找到可统计的 knowledge_points。");
} else {
  dv.table(["知识点", "出现次数"], rows);
}
```

## 三、组卷操作台（基础 / 综合 / 挑战）

> 这一节直接服务于 [[模板-专题练习卷]]。
> 如果只是要筛题，请优先使用页首“推荐入口”里的交互面板。
> 使用方法：
> 1. 先把查询里的 `目标KP` / `专题关键词` 改成当前专题的核心词
> 2. 若是多知识点专题，用 `OR contains(string(knowledge_points), "...")` 继续补条件
> 3. 第一轮通常取 `基础 3-4 题 + 综合 1-2 题`
> 4. 第二轮通常取 `基础 2 题 + 综合 2-3 题 + 挑战 1 题`

### 3.1 基础巩固池（difficulty = 2）

```dataview
TABLE WITHOUT ID
  link(file.path, title) AS 题目,
  year AS 年份,
  module AS 模块,
  question_type AS 题型,
  difficulty AS 难度,
  join(knowledge_points, "、") AS 知识点
FROM "04-题库/真题"
WHERE contains(string(knowledge_points), "目标KP")
  AND difficulty = 2
SORT year DESC, file.name ASC
LIMIT 12
```

### 3.2 综合应用池（difficulty = 3）

### 3.4 多知识点专题模板（可直接改词）

> 适合“一个专题不止一个 KP 名称”的情况，例如化学平衡、晶体结构、酸碱理论。

```dataview
TABLE WITHOUT ID
  link(file.path, title) AS 题目,
  year AS 年份,
  question_type AS 题型,
  difficulty AS 难度,
  join(knowledge_points, "、") AS 知识点
FROM "04-题库/真题"
WHERE (
    contains(string(knowledge_points), "专题关键词A")
    OR contains(string(knowledge_points), "专题关键词B")
    OR contains(string(knowledge_points), "专题关键词C")
  )
  AND difficulty >= 2
  AND difficulty <= 4
SORT difficulty ASC, year DESC
LIMIT 20
```

### 3.6 组卷前快速检查

| 检查项 | 建议 |
|:---|:---|
| 是否全是同一种题型 | 至少混入 2 种题型，避免整张卷只练一种动作 |
| 是否连续 2 题考同一小点 | 尽量错开，避免“一错全错” |
| 是否只有高难题 | 第一轮/第二轮先保留 2-3 道可稳拿分的题 |
| 是否缺少题目来源跨度 | 尽量混合不同届次，避免同一套卷风格过于单一 |
| 是否能回链到 KP | 题目优先保留 `knowledge_points` 完整的条目 |

## 四、有机专项（当前冲刺重点）

> 有机化学是基础要求最后一块拼图，此处单独列出有机相关真题以便集中突破。

### 4.1 有机真题列表

```dataview
TABLE WITHOUT ID
  link(file.name, title) as 题目,
  year as 年份,
  question_type as 题型,
  difficulty as 难度,
  join(knowledge_points, "、") as 核心知识点
FROM "04-题库/真题"
WHERE module = "有机化学"
SORT year DESC, difficulty ASC
```

### 4.2 有机真题涉及的知识点覆盖

```dataview
TABLE WITHOUT ID
  knowledge_points as 知识点,
  count(file.name) as 考查次数,
  join(map(rows, (r) => link(r.file.name, r.title)), "、") as 关联题目
FROM "04-题库/真题"
WHERE module = "有机化学"
FLATTEN knowledge_points
GROUP BY knowledge_points
SORT count(file.name) DESC
```
