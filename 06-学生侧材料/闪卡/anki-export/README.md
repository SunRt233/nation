# Anki CSV 导出说明

本目录存放从 `03-知识点/` 批量导出的 Anki CSV。

> **页面边界（2026-06-15）**：本页只负责“如何从 KP 导出 Anki CSV，以及导入 Anki 时怎么用”。它不是学生侧材料总入口，也不负责决定哪些知识点本周一定要复习。
> - 复习节奏与 `review_cycle` 维护：去 [[00-首页/KP复习清单]]
> - 学生闪卡材料总入口：去 [[06-学生侧材料/闪卡/README]]

## 当前最小工作流

1. 维护 KP 的 `review_cycle` 字段，只使用 `7d / 30d / 90d`
2. 运行根目录脚本：

```powershell
node .\export_anki_csv.js
```

3. 生成两个文件：
   - `anki-kp-cards.csv`：完整导出
   - `anki-kp-cards-sample.csv`：前 30 行样例

## 当前导出字段

| 字段 | 作用 |
|:---|:---|
| `Front` | 卡片正面 |
| `Back` | 卡片背面 |
| `Tags` | 学科 / 模块 / KP / `rc_*` 标签 |
| `Deck` | 初始牌组建议 |
| `Source` | 来源 KP wikilink |
| `ReviewCycle` | 原始复习周期 |
| `NoteType` | `概念卡` / `总结卡` / `易错点卡` |
| `Extra` | 补充说明 |

## 牌组映射

- `7d -> Chem::Weekly`
- `30d -> Chem::Core`
- `90d -> Chem::Review`

## 当前提取规则

- `## 一、定义` -> 概念卡
- `## 四、关键结论` -> 总结卡
- `## 十一、易错点` -> 易错点卡
- 含 `待填充` 的内容自动跳过

## 导入 Anki 时的建议

- 分隔符选择 `,`
- 编码选择 `UTF-8`
- 可将 `Tags` 映射到标签字段
- 可将 `Deck` 作为导入前分流参考，或先统一导入再按标签筛选

## 已知边界

- 当前不处理图片、LaTeX 优化、cloze 挖空、机理 Image Occlusion
- 当前不读取专题页，只读取 `03-知识点/` KP
- 当前更偏“先可用”，不是最终闪卡质量上限
