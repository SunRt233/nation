# KB-Image-Finder · 知识库图片资产检索

## 用途

将 `[[Agent标准作业流程]]` 规则 7 中约 500 字的图片检索流程压缩为 **一步触发、自动执行**。Agent 只需提供主题（如"钙钛矿结构"），skill 自动走完三段式检索并返回可用图片清单。

**核心价值**：
- 避免 Agent 每次配图时重新推理"先查精标表还是 Grep mineru"
- 内置 OCR 噪声黑名单，自动过滤已知陷阱图
- 输出包含嵌入语法 + frontmatter 更新指令，一步到位

---

## 触发条件（满足任一即激活）

| 场景 | 示例用户消息 |
|:---|:---|
| **KP 配图** | "给 `[[钙钛矿结构]]` 配张图" / "这个 KP 需要晶胞图" |
| **备课找图** | "找张 Frost 图的实例" / "需要一张晶体场分裂示意图" |
| **图注笔记** | "给这张图建图注" / "查一下这题的配图在哪" |
| **主动检索** | "图片检索" / "mineru 里有没有 XX 的图" |

---

## 执行步骤

### Step 0: 明确检索主题（~3 秒）

从用户消息中提取**检索主题**。主题应尽量与 KP 标题一致，便于匹配精标表。

| 用户消息 | 提取主题 |
|:---|:---|
| "给 [[钙钛矿结构]] 配张图" | 钙钛矿结构 |
| "找张 SN2 反应的机理图" | SN2反应（或"亲核取代"） |
| "需要晶体场分裂的能级图" | 晶体场分裂 |
| "这张图是哪道题的"（用户已贴图） | 从图文件名或上下文推断 |

若用户消息中无明确主题，**先询问确认**再执行检索。

### Step 1: 一段 · 精标表速查（~5 秒）

读取 `[[00-首页/图片索引]]` §精标表。

- 在精标表中搜索主题（模糊匹配，支持中英文/别名）
- 命中 → 直接提取该行的完整信息，跳到 **Step 3**（视觉验证）
- 未命中 → 进入 **Step 2**（文本检索）

> 精标表字段：`主题 | 路径 | 关联 KP / 真题 | 上下文摘要 | 类型 | 置信度 | 图注笔记`

### Step 2: 二段 · 文本检索路径（~10-20 秒）

#### 2.1 查已知主题检索 hint

读取 `[[00-首页/图片索引]]` §已知主题 → 检索 hint。

若主题在 hint 表中有记录：
- 直接使用推荐的 **Grep 关键词组** 和 **优先搜索目录**
- 跳到 2.3

若无记录：
- 构造关键词组：主题名 + 英文名 + 常见别名/缩写
- 示例：`晶体场分裂` → Grep `晶体场\|分裂能\|t2g\|eg\|CFSE\|10Dq\|配体场`

#### 2.2 Grep 文本命中

```
Grep "关键词组" mineru/ → 列出命中的 .md 文件
```

优先搜索目录（按命中率排序）：
1. `mineru/01-真题/` — 真题原图，通常带题面和完整上下文
2. `mineru/03-教材书籍/` — 教材插图，质量高
3. `mineru/04-其他试题/` — 模拟题/研讨题，专题性强
4. `mineru/02-真题解析/` — 解析中的补充图

#### 2.3 定位图引用

对命中的每个 .md 文件：
```
Grep '!\[' -n → 看图引用所在行号
```

mineru 图引用格式：`![](xxx_images/<hash>.jpg)`

#### 2.4 上下文匹配

读取该 md 文件"图引用行 ± 10 行"范围：
- 图前后的题面/章节标题 = 天然语境
- 若上下文明确写"图 X.Y <主题>"→ 高把握候选
- 若上下文只说"如下图"→ 低把握，需视觉验证

提取 1-3 张候选图，按上下文匹配度排序。

### Step 3: 三段 · 视觉验证（~10-20 秒）

#### 3.1 OCR 噪声黑名单过滤

读取 `[[00-首页/图片索引]]` §⚠ OCR 噪声黑名单。

若候选图路径在黑名单中 → **直接丢弃**，不验证。

#### 3.2 采样验证规则

| 图片来源 | 验证规则 |
|:---|:---|
| **mineru 章节首次接触** | 必须至少 Read 1 张采样验证 |
| **mineru 同章节后续图** | 采样通过后，高把握图可批采用 |
| **真题原图 / 自截图 / Excalidraw** | 不受此规则约束，可直接采用 |
| **低把握**（上下文模糊） | 每张候选都要 Read |

#### 3.3 验证内容

Read 图片后确认：
1. 图内容确实与主题相关（非二维码/版权页/印刷标记）
2. 图质量可用（不模糊、不截断、化学式可辨）
3. 确认后记录置信度：🟢 已视觉验证 / 🟡 仅上下文判断

### Step 4: 输出与沉淀（~5 秒）

#### 4.1 输出候选列表

```markdown
## 🔍 图片检索结果 · {主题}

### 候选图（按推荐度排序）

| 排序 | 路径 | 来源 | 类型 | 置信度 | 建议 |
|:---:|------|------|:---:|:---:|---|
| 1 | `![[mineru/.../xxx.jpg]]` | Weller §5.13 章节首图 | 公式/示意 | 🟢 | **首选**：直接嵌入 |
| 2 | `![[mineru/.../yyy.jpg]]` | 36届初赛第3题 | 真题原图 | 🟢 | 备选：若首选不适用 |
| 3 | ... | ... | ... | 🟡 | 低把握，建议人工确认 |
```

#### 4.2 frontmatter 更新指令

选定图片后，输出对应的 frontmatter 修改：

```yaml
# 如果配图
has_images: true
image_count: 1
images_priority: low
images_note: "Weller §5.13 图5.5 一般形式，纵 vE° 横 N"

# 如果决定不配（如 LaTeX 已足够）
has_images: false
image_count: 0
images_priority: low
images_note: "LaTeX 公式 + Mermaid 决策图已足够，无需图像文件"
```

#### 4.3 图注笔记触发判断

若某张图满足以下条件之一，输出"建议创建图注笔记"：
- 被 ≥2 个 KP 引用（已知或预期）
- 在 ≥2 道真题中出现
- 教材经典插图（如 Weller 图5.5）

图注笔记规范见 `[[Agent标准作业流程]]` §7.Y。

#### 4.4 增量沉淀（Phase 4 时同步做）

本次找到的图 → 写入 `[[00-首页/图片索引]]` §精标表 新增一行。

## 降级与兜底

### 全路径未命中

若一段+二段均未命中：
1. Glob `mineru/*/` 看全目录，挑可能相关的（如教材某章）
2. 进该目录的 md → 搜近义词 / 上位词
3. 仍找不到 → 输出"未在现有图片资产中找到匹配图"，建议：
   - 方案 A：自己造图（Mermaid > 截图 > Excalidraw）
   - 方案 B：标记 `images_priority: high` 待后续补充

### 多个同名主题

若检索主题匹配多个不同概念（如"氧化"可能是有机氧化反应或无机氧化还原）：
- 列出歧义项
- 询问用户确认具体是哪一个
- 或按 KP 的 subject 字段缩小范围

## 输出格式

```markdown
## 🔍 图片检索结果 · {主题}

### 检索路径
- 一段精标表：{命中/未命中}
- 二段文本检索：Grep "{关键词}" → 命中 {N} 个文件
- 三段视觉验证：已验证 {N} 张

### 候选图

{候选表}

### 推荐方案
**首选**：`![[路径]]`（理由：{为什么选这张}）

### frontmatter 更新
```yaml
has_images: true
image_count: 1
images_priority: low
images_note: "{评估理由}"
```

### 后续建议
- {是否需要创建图注笔记}
- {是否需要在 Phase 4 沉淀到精标表}
```

## 与 SOP 的衔接

```
用户说"给XX配图"
    ↓
kb-image-finder Skill 触发
    ↓
自动走完三段式检索 → 输出候选图 + frontmatter 指令
    ↓
Agent 直接 Edit KP 文件嵌入图片、更新 frontmatter
    ↓
Phase 4 归档时：沉淀到精标表 + 必要时创建图注笔记
```

**关键原则**：
- Skill 只做"找图"，不做"嵌入"（嵌入由 Agent 用 Edit 执行）
- 视觉验证发现 OCR 噪声 → 立即写入 `[[图片索引]]` §⚠ OCR 噪声黑名单
- 低把握图（🟡）必须告知用户"建议人工确认"
