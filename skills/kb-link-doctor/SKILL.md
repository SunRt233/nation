# KB-Link-Doctor · 断链检测与修复

## 用途

将 `[[11-模板/脚本-断链检测]]` 中约 150 行的 PowerShell/Node.js 脚本封装为 **一句话触发** 的自动化流程。Agent 只需说"检测断链"，skill 自动完成扫描、分类、修复、报告的全链路。

**核心价值**：
- 避免 Agent 每次手动复制粘贴 PowerShell 脚本
- 内置假阳性过滤，减少人工分析报告时间
- 低风险断链自动修复，无需用户确认

| **定期维护** | "月度断链检测" / "跑一下 link audit" |

## 执行步骤

### Step 1: 全库扫描（~10-20 秒）

收集 vault 中所有 `.md` 文件，提取所有 `[[...]]` wikilink。

```
扫描范围：全库（排除 .obsidian/、.trash/、.claude/）
链接格式：[[目标]]、[[目标|别名]]、[[路径/目标]]、[[路径/目标#标题锚点]]
```

建立索引：
- `allPaths`：所有文件的相对路径（含扩展名）
- `allNames`：所有文件的不含扩展名文件名
- `nameToPaths`：文件名 → 路径列表映射（处理同名文件）

### Step 2: 链接有效性判断（~10 秒）

对每个 wikilink 提取目标（去掉别名、块引用、标题锚点）：

```
原始链接：[[03-知识点/有机化学/Lawesson试剂|试剂]]
提取目标：03-知识点/有机化学/Lawesson试剂
```

判断存在性：
- **带路径链接**（含 `/`）：先精确匹配路径，再回退匹配文件名
- **纯文件名链接**：匹配 `allNames`

### Step 3: 假阳性过滤（~5 秒）

以下链接**不算断链**，直接跳过：

| 类型 | 示例 | 过滤规则 |
|:---|:---|:---|
| **空/占位符** | `[[-]]`、`[[知识点-]]` | 匹配占位符正则 |
| **待建标记** | `[[待建KP]]`、`[[待补充]]` | 含"待建"/"待补充"字样 |
| **考纲编号占位** | `[[01]]`、`[[3-2]]` | 纯数字或考纲编号格式 |
| **.base 文件** | `[[课件.base]]` | 以 `.base` 结尾 |
| **路径前缀链接** | `[[03-知识点/有机化学/试剂]]` | 文件名实际存在，只是带了路径前缀 |
| **图片示例** | skills 文件夹中的示例链接 | 路径含 `skills/` 且为示例 |
| **外部 URL** | 误写成 wikilink 的 http 链接 | 含 `http`/`https` |

### Step 4: 断链分类（~5 秒）

将真断链分为 3 类：

| 类型 | 特征 | 处理方式 |
|:---|:---|:---|
| **A. 可自动修复** | 拼写错误（大小写、单复数、常见变体） | 直接 Edit 修复 |
| **B. 需人工确认** | 目标文件确实不存在，需判断是新建/改名/删除 | 列出清单，等待用户决策 |
| **C. 假阳性残留** | 过滤规则未覆盖的误报 | 标记为待优化，更新过滤规则 |

#### A 类自动修复规则

| 错误模式 | 示例 | 修复为 |
|:---|:---|:---|
| 大小写不匹配 | `[[frost图]]` | `[[Frost图]]` |
| 全角符号 | `[[Frost图　]]`（全角空格） | `[[Frost图]]` |
| 多余空格 | `[[Frost 图]]` | `[[Frost图]]` |
| 常见别名 | `[[马氏规则]]` | `[[Markovnikov规则]]`（若后者存在） |

> ⚠️ **安全原则**：只修复**有且仅有 1 个可能目标**的链接。若存在多个候选（如同名文件在不同目录），标记为 B 类，不自动修复。

### Step 5: 输出报告（~5 秒）

```markdown
## 🔗 断链检测报告 · YYYY-MM-DD HH:MM

### 统计

| 指标 | 数值 |
|:---|---:|
| 扫描文件数 | {N} |
| 总 wikilink 数 | {N} |
| 有效链接 | {N} |
| 占位跳过 | {N} |
| **真断链** | **{N}** |
| 已自动修复 | {N} |
| 需人工确认 | {N} |

### 已自动修复（A 类）

| 源文件 | 原链接 | 修复后 |
|:---|:---|:---|
| `03-知识点/化学原理/Frost图.md` | `[[frost图]]` | `[[Frost图]]` |
| ... | ... | ... |

### 需人工确认（B 类）

| 源文件 | 断链 | 建议操作 |
|:---|:---|:---|
| `04-专题/专题总索引.md` | `[[专题-气体计算]]` | 🔴 文件不存在 → 确认是待建专题还是拼写错误 |
| `04-专题/专题总索引.md` | `[[专题-化学平衡综合计算]]` | 🔴 文件不存在 → 同上 |
| ... | ... | ... |

### 假阳性残留（C 类）

以下链接被判定为断链，但实际可能是合法引用：

| 源文件 | 链接 | 原因 |
|:---|:---|:---|
| ... | ... | 过滤规则未覆盖 |

> 如确认是假阳性，请在 `[[11-模板/脚本-断链检测]]` 中更新占位符过滤规则。

---

*报告由 kb-link-doctor 自动生成*
```

### Step 6: 归档（Phase 4 时）

- 报告保存到 `11-模板/broken_links_report_YYYYMMDD.txt`
- 或在 `09-审计报告/` 下创建 `YYYY-MM-DD-断链检测.md`
- 工作日志中记录：本次修复数量、剩余待确认数量

## 特殊场景处理

### 场景 A：批量重命名后检测

用户说"我刚把一批 KP 从'旧名'改成'新名'，检查下断链"。

处理：
1. 执行常规扫描
2. 额外输出"重命名相关断链"分类（源文件含"旧名"的链接）
3. 若旧名和新名有明确映射关系，可直接批量替换

### 场景 B：指定目录检测

用户说"只检查 03-知识点/ 里的断链"。

处理：缩小扫描范围到指定目录，但**链接目标仍需全库匹配**（因为 wikilink 可以跨目录指向）。

### 场景 C：断链数异常激增

如果本次断链数比上次检测增加 >50%：
- 输出 ⚠️ 警告："断链数异常增加，可能由批量移动/重命名导致"
- 建议启用 SOP 完整模式，人工核查后再批量修复

## 与 SOP 的衔接

```
用户说"检测断链"
    ↓
kb-link-doctor Skill 触发
    ↓
自动扫描 → 过滤假阳性 → 分类 → 自动修复 A 类
    ↓
输出报告（含 B 类待确认清单）
    ↓
用户对 B 类给出决策 → Agent 执行修复
    ↓
Phase 4 归档：报告存入 09-审计报告/ + 更新工作日志
```

**关键原则**：
- Skill 只做"检测 + 低风险自动修复"，不做大规模自动重命名
- 任何涉及 >5 个文件的修改，必须在报告中明确列出并建议用户确认
- 修复后的链接必须使用正斜杠 `/`（跨平台兼容）
