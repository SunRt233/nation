# KB-KP-Creator · 知识点快速创建

## 用途

将 `[[Agent标准作业流程]]` Agent 2（知识点填充）中"创建新 KP"的繁琐流程封装为 **一句话触发**。Agent 只需说"建一个关于XX的知识点"，skill 自动完成模板读取、frontmatter 填充、目录选择、文件创建的全链路。

**核心价值**：
- 避免 Agent 每次手动复制粘贴模板、改文件名、填 frontmatter
- 自动选择正确目录（基于学科/subject 映射）
- 自动避免命名冲突（同名检测 + 自动提示）
- 批量创建支持（一次创建 N 个 KP）

---

## 触发条件（满足任一即激活）

| 场景 | 示例用户消息 |
|:---|:---|
| **单点创建** | "建一个 `[[Jahn-Teller效应]]` 的知识点" / "创建 KP：De Mayo反应" |
| **批量创建** | "建5个有机反应的知识点" / "批量创建以下 KP：A、B、C" |
| **资料驱动** | "从 Clayden 第5章提炼一个 KP" / "基于真题创建一个知识点" |
| **断链驱动** | "题库里有红链指向 `[[扩散动力学]]`，建一下这个 KP" |

---

## 执行步骤

### Step 0: 提取 KP 信息（~3 秒）

从用户消息中提取以下字段：

| 字段 | 提取方式 | 示例 |
|:---|:---|:---|
| **名称** | 从 `[[名称]]` 或引号中提取 | "建 `[[扩散动力学]]`" → 扩散动力学 |
| **学科** | 用户显式声明 / 从上下文推断 / 询问确认 | "有机化学的知识点" → 有机化学 |
| **模块** | 用户显式声明 / 默认留空 | "化学平衡模块" → 化学平衡 |
| **状态** | 默认"框架"；若用户说"填充好"→"初稿" | - |
| **重要性** | 用户声明 / 默认 3 | "很重要" → 5 |
| **来源** | 从用户消息中提取教材/书籍/真题来源 | "从 Clayden 第5章" → 来源 |

若用户消息中**无明确学科**，**必须询问确认**后再执行。学科是目录路由的关键输入。

### Step 1: 前置检查（~5 秒）

#### 1.1 目录映射

基于 `subject` 字段映射到目标目录：

| subject | 目标目录 |
|:---|:---|
| 有机化学 | `03-知识点/有机化学/` |
| 化学原理 | `03-知识点/化学原理/` |
| 无机和结构化学 | `03-知识点/无机和结构化学/` |
| 分析化学 | `03-知识点/分析化学/` |
| 综合 | `03-知识点/综合/` |
| 决赛要求 | `03-知识点/决赛要求/`（需进一步判断子模块） |

> 决赛要求子模块判断：
> - 物理化学相关 → `决赛要求/物理化学深化/`
> - 结构/配位相关 → `决赛要求/结构与配位深化/`
> - 金属有机/超分子 → `决赛要求/金属有机与超分子/`

#### 1.2 命名冲突检测

```
检查：03-知识点/<目录>/<名称>.md 是否已存在
- 已存在 → 输出冲突警告，询问：覆盖 / 跳过 / 改名
- 不存在 → 继续
```

#### 1.3 目录就绪检查

```
检查：目标目录是否存在
- 存在 → 继续
- 不存在 → 创建目录（极少发生，03-知识点结构已稳定）
```

### Step 2: 读取模板（~3 秒）

读取 `[[11-模板/模板-知识点]]`，提取：
- frontmatter 模板（YAML）
- 正文结构模板（Markdown）
- 当前 `template_version`

### Step 3: 生成 KP 文件（~5 秒）

#### 3.1 frontmatter 填充规则

基于模板，按以下规则填充字段：

| 字段 | 填充规则 |
|:---|:---|
| `title` | KP 名称 |
| `aliases` | `[]`（用户未提供时）或用户提供的别名列表 |
| `type` | 固定 `知识点` |
| `subject` | 用户指定的学科 |
| `module` | 用户指定的模块，或根据名称推断 |
| `submodule` | 用户指定，或留空 |
| `syllabus_stage` | 默认 `基础`；决赛内容可改为 `拔高` |
| `status` | 默认 `框架`；用户说"填充"则 `初稿` |
| `difficulty` | 默认空；用户指定则填入 |
| `importance` | 默认 `3`；用户指定则覆盖 |
| `updated` | 今天日期 `YYYY-MM-DD` |
| `template_version` | 当前模板版本 |
| `sources` | 用户提供的来源，或 `[]` |
| `tags` | 默认 `[化竞]` + 学科标签 |
| `has_images` | 默认 `false` |
| `image_count` | 默认 `0` |
| `images_priority` | 默认 `low` |
| `images_note` | 默认 `"待评估"` |
| `key_images` | 默认 `[]` |

#### 3.2 正文生成规则

保留模板的完整结构，将 `"标题"` 替换为 KP 名称。其余章节保留占位符 `(待填充)`。

## 批量创建模式

用户说"批量创建以下 KP"或"建5个知识点"时：

1. 列出所有 KP 名称
2. 询问用户确认学科/模块（若未指定）
3. **逐个创建**，每创建一个输出一行确认
4. 最后汇总输出创建清单

> ⚠️ **安全原则**：批量创建 >5 个时，先创建 1 个样例让用户确认格式，再继续批量。

## 特殊场景处理

### 场景 A：用户只说"建知识点"但没给名称

```
用户："建个知识点"
Agent："请提供知识点名称，例如：建一个 `[[扩散动力学]]` 的知识点"
```

### 场景 B：文件已存在

```
检测到 03-知识点/化学原理/扩散动力学.md 已存在。
请选择：
- [ ] 覆盖（用新模板替换）
- [ ] 跳过（保持现有文件）
- [ ] 改名（创建为 扩散动力学-v2.md）
```

### 场景 C：学科不明确

```
"不对称合成" 可以属于有机化学或决赛要求。
请选择学科：
- [ ] 有机化学（基础）
- [ ] 决赛要求/金属有机与超分子（拔高）
```

### 场景 D：从资料提炼创建

用户说"从 Clayden 第41章提炼一个不对称合成的 KP"：

处理：
1. 先执行 kb-kp-creator 创建框架
2. 在 sources 中填入 `[[07-资料提炼/书籍提炼/提炼-Clayden-第41章-不对称合成]]`
3. 提示用户：框架已创建，接下来可调用资料提炼 Agent 填充内容

## 与 SOP 的衔接

```
用户说"建个知识点"
    ↓
kb-kp-creator Skill 触发
    ↓
提取信息 → 前置检查 → 读取模板 → 生成文件
    ↓
输出创建结果（含路径 + frontmatter 摘要）
    ↓
Phase 4 归档：工作日志记录 + 更新知识点总索引（Dataview 自动）
```

**关键原则**：
- Skill 只做"创建框架文件"，不做内容填充（内容填充由 Agent 2 执行）
- 任何涉及覆盖现有文件的修改，必须用户确认
- 创建后的 KP 默认 status 为"框架"，不冒充"已填充"
- 批量创建 >5 个时，先创建样例确认格式
