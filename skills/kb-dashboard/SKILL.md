# KB-Dashboard · 知识库状态仪表盘

## 用途

在 Agent 动手执行任务之前，先用 **<30 秒** 时间建立对知识库当前状态的全局认知。替代 `[[Agent标准作业流程]]` Phase 0 的默认模式（读 1 个文件）甚至完整模式（读 5-6 个文件）。

**核心价值**：
- 避免 Agent "盲飞" —— 不知道全库已填充 603/603 还在问"要建多少骨架"
- 自动判断意图路由 —— 用户说"优化一下"应该走维护型 Agent，说"备个课"应该走使用型 Agent
- 给出下一步推荐 —— 基于战略方向和当前阻塞项，建议最高 ROI 的工作

---

## 触发条件（满足任一即激活）

| 场景 | 示例用户消息 |
|:---|:---|
| **启动速查** | Agent 新会话首条消息涉及知识库 |
| **状态询问** | "状态报告" / "知识库现在什么情况" / "今天该做什么" |
| **模糊推进** | "继续推进" / "开始工作" / "干活" / "下一步" |
| **任务前加载** | 用户在给出具体任务前，Agent 主动加载上下文 |

---

## 执行步骤

### Step 1: 速读核心文件（~10 秒）

按以下顺序读取，**只读一遍**：

| 顺序 | 文件 | 读取重点 |
|:---:|------|---------|
| 1 | `[[00-首页/状态摘要]]` | ① 当前关键状态表格（机器可读段落）<br>② 当前会话遗留任务<br>③ 体系版本表（SOP/模板/Agent 提示词版本）<br>④ 🔥 高优 Follow-up（如 Dataview 结果不可读，读表格后的文字描述） |
| 2 | `[[00-首页/Agent战略方向]]` | ① 当前所处阶段表格<br>② 工作大方向 A-E 的优先级<br>③ 备课进度总览（五轮框架状态）<br>④ Agent 默认行为决策树 |

> **速读技巧**：跳过 Dataview 代码块本身，只读代码块前后的文字解释和表格。Dataview 查询在 Claude 环境不可执行，但状态摘要的"当前关键状态"段落已用文字总结了 Dataview 结果。

### Step 2: 提取关键状态指标（~5 秒）

从 Step 1 读取的内容中，提取以下字段：

```
## 提取清单（内部使用，不输出给用户）

[库存]
- kp_total:      从"当前关键状态"表格读取（如 603）
- kp_filled:     同上（如 603，若全填充则=total）
- syllabus_gap:  从"考纲缺口"读取（如 0）
- phase:         从"阶段"读取（如"使用期+深化期"）

[健康]
- broken_link_count: 从"当前关键状态"或最近审计报告读取
- high_priority_images: 从"高优待补"读取（如 0）
- pending_followup: 从 🔥 段落统计未完成项数

[版本]
- sop_version: 从体系版本表读取（如 v1.4.4）
- kp_template_version: 同上（如 v1.3）

[战略]
- current_focus: 从"当前重心"读取（如"内容深化 + 课件产出"）
- next_direction: 从"下一步"读取（如"路线 D：v3 审计"）
- lesson_progress: 从"备课进度总览"读取（如"第一轮 1/10 课件产出"）

[阻塞]
- blockers: 从"阻塞"字段读取列表
```

### Step 3: 判断用户意图（~5 秒）

将用户的原始消息映射到 6 类入口路由：

| 用户用词/场景 | 意图分类 | 路由 |
|:---|:---|:---|
| "提炼"/"填充"/"创建"/"建知识点" / 资料导入 | 📥 构建 | `[[Agent提示词-知识库构建]]` 1-6 |
| "优化"/"改进"/"完善"/"合并"/"弃用"/"扫一遍" | 🔧 维护 | `[[Agent提示词-知识库维护]]` 7-9 |
| "备课"/"做大纲"/"准备一节课"/"整理讲义" | 📚 使用 | `[[Agent提示词-知识库使用]]` 10-11 |
| "什么是 X"/"X 怎么反应"/"为什么 X" | 💬 提问 | 不走 SOP，直接基于 KB 回答 |
| "改 SOP"/"调整模板"/"重新设计" | 🛠️ 元讨论 | 走完整 5-Phase，Phase 1 必须用户确认 |
| 无明确任务 / "继续推进" / "下一步" | 🤖 自主推进 | 见 Step 4 默认行为 |

> 若用户消息同时包含多类关键词，取**最具体**的一类。例如"备课时我发现某 KP 内容太浅需要优化" → 先走 📚 使用（备课），在备课过程中如发现 KP 问题，再记录 follow-up 后续走 🔧 维护。

### Step 4: 生成输出（~10 秒）

按下方**输出格式模板**输出。若用户意图为 🤖 自主推进，额外输出"推荐下一步"。

---

## 输出格式模板

```markdown

### 一、库存概览

| 指标 | 数值 | 状态 |
|:---|:---|:---:|
| 知识点总数 | {kp_total} | {kp_total == kp_filled ? "✅" : "🔧"} |
| 已填充 | {kp_filled}/{kp_total} ({round(kp_filled/kp_total*100)}%) | - |
| 考纲缺口 | {syllabus_gap} | {syllabus_gap == 0 ? "✅ 闭环" : "🔴 待补"} |
| 高优待补图片 | {high_priority_images} | {high_priority_images == 0 ? "✅ 清零" : "🟡 待补"} |
| SOP 版本 | {sop_version} | - |

> **阶段判断**：{phase}。核心矛盾：{从战略方向读取的核心矛盾描述}

---

### 二、当前重心与阻塞

**工作重心**：{current_focus}

**阻塞项**：
{blockers 列表，若无则写"无明确阻塞"}

---

### 三、高优 Follow-up（未完成）

{从状态摘要提取当前会话遗留任务，最多列 5 条，按优先级排序}

---

### 四、入口路由

用户意图判断：**{意图分类}**

> {路由说明：走哪个 Agent 提示词文件，是否需要完整 5-Phase}

**推荐 Agent**：`{对应的提示词文件和章节}`

**前置检查**：
- [ ] 源文件是否存在
- [ ] 目标目录是否就绪
- [ ] 模板版本是否匹配（当前模板 v{kp_template_version}）

---

### 五、下一步推荐（仅当用户无明确任务时输出）

基于 `[[Agent战略方向]]` §三、Agent 默认行为：

```
1. 高优 Follow-up 检查 → {pending_followup > 0 ? "有 {n} 项未完成，建议优先处理" : "无未完成项"}
2. 高优待补图片检查 → {high_priority_images > 0 ? "有 {n} 个 KP 待补图" : "已清零"}
3. 备课进度推进 → {lesson_progress}，下一课件：{从备课进度表读取下一个未完成专题}

```

**最高 ROI 推荐**：{从上述 4 步中选出当前最该做的 1 项，一句话说明理由}
```

## 特殊场景处理

### 场景 A：用户只说"继续推进"或"开始工作"

输出完整 5 节内容（含第五节"下一步推荐"），然后**等待用户确认**再执行具体任务。不要擅自启动大规模操作。

### 场景 B：用户已有明确任务（如"把第三章提炼一下"）

跳过第五节。在第四节"入口路由"中明确映射到对应 Agent，然后**直接进入 SOP Phase 1**（制定执行计划）。此时 Dashboard 的作用已完成，不要让 Dashboard 成为阻塞。

### 场景 C：SOP 版本 mismatch

如果状态摘要中的 `sop_version` 与 skill 内部预期的版本不一致（如 skill 写时针对 v1.4，实际读到 v1.6），在输出中追加 ⚠️ 警告：

> ⚠️ **版本警告**：本 skill 基于 SOP v{skill_version} 设计，检测到当前 SOP 为 v{actual_version}。部分规则可能已变更，建议 Agent 在 Phase 0 启用**完整模式**读取全部 5 个文件。

### 场景 D：Dataview 不可读时的降级

如果状态摘要中某些信息完全依赖 Dataview 渲染结果（如库存表格只有代码块没有文字总结），而代码块不可读：
1. 用 `Glob` 扫描 `03-知识点/` 下的 `.md` 文件数作为 kp_total 的近似值
2. 用 `Grep "status: 已填充" 03-知识点/` 的计数作为 kp_filled 的近似值
3. 在输出中标注："📊 部分数据为近似值（Dataview 不可执行，已用文件扫描替代）"

## 与 SOP 的衔接关系

```
用户消息 → kb-dashboard Skill 触发
    │
    ├─ 输出"状态快照 + 入口路由"
    │
    └─ 根据意图分类，进入对应路径：
        ├─ 📥/🔧/📚 → 进入 [[Agent标准作业流程]] Phase 1（制定计划）
        ├─ 💬 → 不走 SOP，直接基于 KB 回答
        ├─ 🛠️ → 进入完整 5-Phase，Phase 1 必须用户确认
        └─ 🤖 → 输出"下一步推荐"，等待用户确认
```

**关键原则**：
- Dashboard Skill **不替代** SOP，只替代 Phase 0 的上下文加载
- Dashboard 完成后，Agent 仍需按 SOP 走 Phase 1-4
- Dashboard 的输出应写入工作日志（Phase 4 归档时作为"本次 session 起始状态"引用）

- **版本**：v1.0（基于 SOP v1.4.4 + 战略方向 v1.0）
