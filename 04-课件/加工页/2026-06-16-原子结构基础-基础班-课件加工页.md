## 4. 抽料规则

### 4.0 默认优先级

1. 教材主干与备课框架：决定“这节课讲什么”
2. KP / 专题现成资产：负责结构化复用
3. 网课资料：负责补讲法、类比、易错提醒
4. 题库 / 真题链：负责把内容坐实为例题、变式、练习
5. 图片资产：负责理解增益与投影支持

> 考纲主要用于校边界，而不是机械删教材内容。教材稳定讲、课堂有价值、又不明显过偏的内容，可以保留，但要写清取舍理由。

### 4.1 教师稿从哪里抽

- 课时分配、认知台阶、媒介分工、边界决策：优先取自备课大纲
- 课堂切入语、冲突设计、过渡语、反常点讲法：优先参考两份教学逻辑提炼
- 若备课大纲与讲义表述冲突：
  - 范围与深度以备课大纲为准
  - 学生可见表达与例题组织以现役讲义为准

### 4.2 学生讲义从哪里抽

- 学习目标、知识内容、例题、易错点、作业：优先取自现役新授课
- 教材上稳定出现、且在本节边界内的轨道图像、能级图、周期律反常解释，应补进讲义正文，不要只保留标题
- 网课资料中课堂表达特别清楚的历史转折语句、类比、易错提醒，可转写为讲义说明或旁批
- 题库 / 真题链中已经验证有代表性的题目，应至少抽 1 组“例题 + 变式 / 易错对照”
- 每个核心板块默认过一遍“五件套”：定义 / 原理或推导 / 例题 / 易错点 / 练习
- 不得写入教师侧边界语句

### 4.3 PPT 从哪里抽

- 历史时间线、轨道图、能级图、电子排布规则总表、周期律反常表、例题题干、收束页：可从现役讲义与专题页抽取
- 长推导、完整波函数背景、定量深化：默认保留讲义或板书，不一次性塞进 PPT
- 一页只承载一个核心任务：模型、图景、规则、例题、小结 其一
- 不把讲义段落原封不动贴进 PPT；能压成图、表、流程、短句就压
- PPT 需要的图片优先来自教材图和专题页已有 `key_images`

## 6. 样式与技术约束

- 统一遵循 [[11-模板/模板-课件产出规范]]
- 关键公式与符号优先用 Word 原生数学对象（OMML）
- `1s² / 2p⁶ / 3d⁵ / Fe³⁺ / Cu⁺` 等上下标必须标准化，不能保留伪下标
- 原子轨道、量子数等短表达优先用文本精排，不为简单上标下标滥用图片

- Windows 渲染与导出默认走固定入口脚本

## 7. 产出顺序

1. 先冻结主真相源，不在导出阶段反向大改正文
2. 先出教师上课大纲 `.docx`
3. 再出学生讲义 `.docx`
4. 最后出投影课件 `.pptx`
5. 每次导出后都做 render / 预览检查，再记录字体、图示、上下标、分页、页面负载问题
6. 修完问题后，把“本轮修了什么、还剩什么”写回本加工页
