# 课件总入口

> 本目录存放所有备课产出。课件体系采用 **三件套架构**：
>
> ```
> 备课大纲（教师：教学设计的施工蓝图）
>   ├──→ 新授课（课堂：90min精炼跟随讲义）
>   ├──→ 学生讲义（自学：教材级参考书）
>   └──→ 速查卡（快速：决策树/公式速查）
> ```
>
> **三层的关系**：备课大纲是教学设计的"施工蓝图"，新授课是课堂上跟随90min节奏的精炼讲义，学生讲义是供学生课前预习/课后深化的教材级参考书，速查卡是考前/做题时的快速工具。
>
> **使用顺序**：从 [[首页]] 或 [[状态摘要]] 进入 → 选择专题/知识点 → 用 [[Agent提示词-知识库使用]] Agent 10 生成课件 → 落盘到对应子目录 → 授课后填写复盘。
>
> 2026-06-18 起补充：跨专题查看当前产出状态时，优先看 [[专题产出总览看板]]，不要再靠翻工作日志逐条确认。

## 落盘规范

### 文件命名
```
<课型>/<YYYY-MM-DD>-<主题>.md
```
示例：`新授课/2026-05-20-晶体结构计算.md`

### frontmatter 必填字段
- `lesson_type`: 新授课 / 复习课 / 习题课
- `subject`: 化学原理 / 无机和结构化学 / 分析化学 / 有机化学 / 决赛要求
- `audience_level`: 基础 / 提高 / 决赛
- `duration`: 课时长度（分钟）
- `referenced_kps`: [引用的知识点列表]
- `start_date`: 授课日期

### 复盘联动
授课后 24h 内，在课件末尾的「复盘记录」段填写，或另存为 `已复盘/<日期>-<主题>-复盘.md`。

### 导出联动
若需要把 Markdown 成稿继续产出为 `.docx / .pptx`，先建一张 [[加工页/README|课件加工页]]，再调用 `docx` / `pptx` 技能或对应脚本，避免直接从正文页自由发挥式导出。
